## 1. エグゼクティブサマリー

ITシステム開発・構築サービスを提供する企業において、日常的に行われる業務をClaude Code Skillsとして体系化・自動化するプロジェクトです。

現在、IT企業の多くの業務は属人化しており、品質にばらつきがあります。ドキュメント作成、コードレビュー、テスト設計など、本来は標準化できる業務においても、個人のスキルと経験に依存しています。

本プロジェクトでは、ITシステム開発のバリューチェーン全体（プリセールスから運用保守まで）を対象に、**45の専門スキル**をClaude Code Skillsとして開発・実装します。これにより、業務効率の大幅な向上（30-70%の時間短縮）と品質の標準化を実現します。

**プロジェクト期間**: 18ヶ月（6フェーズ）
**総投資額**: 推定 180,000,000円（内製リソース換算）
**期待ROI**: 180%（5年間累計）
**投資回収期間**: 3.7年（標準シナリオ）

> **注記**: 投資回収期間は標準シナリオ（年間削減効果48,200,000円）に基づく。詳細は02_skills_inventory.md「6.2 投資回収シナリオ」参照。

## 2. プロジェクトの背景と目的

### 2.1 プロジェクトの背景

#### 現在の課題

**属人化の問題**:
- 提案書作成: ベテラン社員のみが高品質な提案書を作成可能
- 見積もり: 見積もり精度が担当者により大きく異なる（誤差 ±30%）
- コードレビュー: レビュー品質がレビュアーのスキルに依存
- テスト設計: テストケース網羅率がテスターにより異なる

**効率性の問題**:
- ドキュメント作成に全体工数の25-35%を消費
- 類似作業の繰り返しによる非効率
- ナレッジの蓄積・活用が不十分

**品質のばらつき**:
- プロジェクトごとの品質管理レベルの差異
- 標準化されたプロセスの不在
- レビュー漏れ、テスト漏れの発生

#### ビジネス上の機会

- **AIによる業務自動化**: Claude等のLLMを活用した業務自動化技術の成熟
- **競合優位性**: 先行してAI自動化を導入することで競争力強化
- **人材不足への対応**: 高度スキル人材の不足をAIで補完
- **スケーラビリティ**: 人員増加なしでの事業拡大が可能

### 2.2 プロジェクトの目的

1. **業務効率化**: 主要業務の処理時間を30-70%短縮
2. **品質標準化**: PMBOK、BABOK、ISO/IEC 29148等の国際標準に準拠した成果物の自動生成
3. **属人化解消**: スキルの明文化・システム化によりナレッジを組織資産化
4. **スケーラビリティ向上**: 人員比例ではない事業成長の実現
5. **競争力強化**: AIファーストの開発プロセスによる差別化

## 7. 期待効果

### 7.1 定量的効果

| 業務領域 | 現状工数 | 削減率 | 年間削減効果 |
|---------|---------|--------|-------------|
| 提案書作成 | 40時間/件 | 50% | 8,000,000円 |
| 見積もり作成 | 16時間/件 | 60% | 4,800,000円 |
| 要件定義書作成 | 80時間/PJ | 40% | 6,400,000円 |
| 設計書作成 | 120時間/PJ | 35% | 8,400,000円 |
| テストケース作成 | 60時間/PJ | 55% | 6,600,000円 |
| コードレビュー | 20時間/Sprint | 45% | 7,200,000円 |
| プロジェクト報告 | 8時間/週 | 60% | 4,800,000円 |
| バグ分析 | 10時間/リリース | 50% | 2,000,000円 |
| **合計（年間）** | - | - | **48,200,000円** |

#### 算出前提

上記の年間削減効果は以下の前提に基づいて算出しています：

| 項目 | 値 | 根拠 |
|------|-----|------|
| **時間単価** | 5,000円/時間 | エンジニア平均人件費（年収800万円÷1,600時間） |
| **年間件数（提案書）** | 80件 | 営業案件数の実績ベース |
| **年間件数（見積もり）** | 100件 | 見積依頼数の実績ベース |
| **年間PJ数** | 20件 | 受注プロジェクト数の実績ベース |
| **スプリント数（年間）** | 200回 | 10PJ × 20スプリント/年 |
| **リリース数（年間）** | 40回 | 20PJ × 2回/年 |

**計算例（提案書作成）**:
```
現状工数: 40時間/件
削減率: 50%
削減時間: 40 × 50% = 20時間/件
年間件数: 80件
年間削減効果: 20時間 × 80件 × 5,000円 = 8,000,000円
```

> **注記**: 上記前提値は標準シナリオの想定値であり、実際のパイロットプロジェクトで検証・調整が必要です。削減率はスキル導入後3ヶ月の安定稼働時の目標値です。

### 7.2 定性的効果

- **品質向上**: 標準化による成果物品質の安定化
- **ナレッジ蓄積**: 組織知の体系化と継承
- **人材育成**: AIアシストによるジュニア人材の早期戦力化
- **顧客満足度向上**: 迅速な対応と高品質な成果物
- **従業員満足度向上**: 単純作業からの解放、創造的業務への集中

## 付録A: スキル設計標準（初期要件ドラフト）

フェーズ1「1.1.3.1 スキル設計標準」で詳細化する設計標準の初期要件：

### A.1 必須構成要素

| 要素 | 必須 | 説明 |
|------|------|------|
| SKILL.md | ◎ | YAMLフロントマター（name, description）必須 |
| scripts/ | ○ | 自動化スクリプト（該当する場合） |
| references/ | ○ | 参照ドキュメント（ガイドライン、API仕様等） |
| assets/ | ○ | テンプレート、ボイラープレート |

### A.2 SKILL.md標準構成

```markdown
---
name: skill-name-kebab-case
description: 50-200文字の説明。いつ使うか（トリガー条件）を含める。
---

# [スキル名]

## When to Use
## Workflow
## Examples
## References（該当する場合）
```

### A.3 品質基準

- `quick_validate.py`を全スキルがパスすること
- 日本語/英語の言語切り替えをサポート（ユーザー言語検出）
- エラー処理：明確なエラーメッセージを出力

> **注記**: 本ドラフトはフェーズ1で「スキル設計標準書」として詳細化される。

## 付録B: CI/CDパイプライン要件（初期要件ドラフト）

フェーズ1「1.1.4.2 CI/CDパイプライン」で構築するパイプラインの初期要件：

### B.1 自動化対象

| ステージ | トリガー | 処理内容 |
|---------|---------|---------|
| **Lint** | Push/PR | SKILL.md構文チェック、Pythonリンター |
| **Validate** | Push/PR | quick_validate.py実行 |
| **Package** | Tag/Release | スキルZIPパッケージ生成 |
| **Publish** | Manual/Release | カタログ更新、配布サイトへのデプロイ |

### B.2 技術スタック候補

- **CI/CDツール**: GitHub Actions（優先）またはGitLab CI
- **リンター**: ruff（Python）、markdownlint（Markdown）
- **パッケージング**: package_skill.py（既存スクリプト活用）

### B.3 成功基準

- すべてのPRがCI通過なしにマージ不可
- リリースタグ作成で自動パッケージング
- パイプライン実行時間: 5分以内

> **注記**: 本ドラフトはフェーズ1で「CI/CD設定」として実装される。
