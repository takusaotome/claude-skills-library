# Pricing Strategist（価格戦略設計）

## Overview

This skill helps design optimal pricing strategies for IT services and consulting businesses. It guides you through pricing model selection, value-based pricing implementation, price elasticity analysis, and price testing design to maximize revenue and profitability.

**Primary language**: Japanese (日本語)
**Output format**: Markdown

**Supported Service Types**:
- MSP（マネージドサービス）
- IT Consulting（ITコンサルティング）
- SaaS（ソフトウェア・アズ・ア・サービス）
- System Development（システム開発・受託開発）
- Hardware Sales（ハードウェア・IT機器販売）

Use this skill when:
- 新製品・サービスの価格設定を行う
- 価格モデル（Cost-plus, Value-based, Competitive, Subscription）を比較・選定する
- 顧客の価格感度を分析したい（価格弾力性分析）
- 価格テスト（A/B, コンジョイント分析）を設計する
- 価格改定の根拠を作成する
- 競合価格分析と自社ポジショニングを行う

## Core Workflows

1. **Pricing Model Selection**: サービス特性に最適な価格モデルを選定
2. **Value-Based Pricing**: 顧客価値に基づく価格設定
3. **Price Elasticity Analysis**: 価格感度測定と最適価格点の特定
4. **Price Test Design**: A/B・コンジョイント分析の設計
5. **IT Services Pricing Strategy**: 業種特化の価格戦略設計
6. **Pricing Documentation**: 価格戦略書・提案書の作成

## Workflow 1: Pricing Model Selection and Comparison（価格モデル選定と比較）

### Step 1: Service Classification（サービス分類）

Classify the service to determine appropriate pricing models:

**サービスタイプ別分類**:

| サービスタイプ | 特徴 | 推奨価格モデル |
|--------------|------|--------------|
| コモディティ化インフラ | 差別化困難、標準化 | Cost-plus, Competitive |
| 戦略コンサルティング | 高付加価値、カスタム | Value-based |
| マネージドサービス | 継続的、予測可能 | Subscription |
| SaaS | スケーラブル、セルフサービス | Tiered Subscription |
| 受託開発 | プロジェクト型 | Fixed-price, T&M with cap |
| ハードウェア | 製品販売 | Cost-plus, Competitive |

### Step 2: Pricing Model Analysis（価格モデル分析）

For each candidate model, analyze using `references/pricing_models_guide.md`:

#### Cost-Plus Pricing（コストプラス）
```
価格 = 直接コスト + 間接コスト配賦 + 目標マージン

例:
直接コスト: 800,000円（人件費500K + 外注200K + 経費100K）
間接コスト配賦: 200,000円（25%）
目標マージン: 300,000円（30%）
価格: 1,300,000円
```

**適用場面**:
- 競争入札
- コモディティサービス
- 原価構造が明確

#### Value-Based Pricing（バリューベース）
```
価格 = 顧客が得る価値 × 価値取得率（10-30%）

例:
年間コスト削減効果: 50,000,000円
リスク軽減価値: 10,000,000円
顧客価値合計: 60,000,000円
価値取得率: 15%
価格: 9,000,000円
```

**適用場面**:
- 戦略コンサルティング
- DXプロジェクト
- 差別化された専門サービス

#### Competitive Pricing（競合ベース）
```
価格 = 競合価格 × ポジショニング係数

例:
競合平均価格: 1,000,000円
自社ポジショニング: プレミアム（1.2倍）
価格: 1,200,000円
```

**適用場面**:
- 成熟市場
- 価格比較が容易な市場
- 新規参入時

#### Subscription Pricing（サブスクリプション）
```
月額 = （顧客生涯価値 × 目標利益率）/ 平均契約期間

例:
顧客生涯価値: 3,600,000円
目標利益率: 40%
平均契約期間: 36ヶ月
月額: 40,000円
```

**適用場面**:
- SaaS
- MSP
- 継続的サービス提供

### Step 3: Hybrid Model Design（ハイブリッドモデル設計）

Combine models for optimal pricing:

**ハイブリッドパターン例**:

| パターン | 構成 | 用途 |
|---------|------|------|
| Base + Variable | 固定月額 + 従量課金 | MSP with usage peaks |
| Fixed + Success Fee | 固定報酬 + 成功報酬 | コンサル with outcomes |
| License + Services | ライセンス料 + 導入支援 | SaaS + Professional Services |
| T&M with Cap | 時間単価 + 上限設定 | 要件不確定な開発 |

### Step 4: Model Comparison Matrix（モデル比較マトリクス）

Create comparison for decision-making:

| 評価軸 | Cost-plus | Value-based | Competitive | Subscription |
|-------|-----------|-------------|-------------|--------------|
| 収益予測性 | 高 | 中 | 中 | 高 |
| 利益率ポテンシャル | 中 | 高 | 低〜中 | 中〜高 |
| 営業難易度 | 低 | 高 | 中 | 中 |
| 顧客受容性 | 高 | 要説明 | 高 | 中 |
| スケーラビリティ | 低 | 中 | 中 | 高 |

## Workflow 2: Value-Based Pricing Implementation（バリューベース価格設定）

### Step 1: Value Discovery（価値発見）

Use value discovery questions from `references/value_based_pricing_methodology.md`:

**価値発見の質問フレームワーク**:

#### ビジネスインパクト質問
1. この問題が解決されない場合、年間どれくらいのコストが発生しますか？
2. 現在の業務プロセスに何人が関わっていますか？時間はどれくらい？
3. この課題によって失われている売上機会はありますか？
4. 競合に対する遅れはどの程度ですか？それは収益にどう影響しますか？

#### 現状コスト分析
```
現状コスト要素:
- 人件費（作業時間 × 時給）: ___円/年
- システム運用コスト: ___円/年
- エラー・手戻りコスト: ___円/年
- 機会損失: ___円/年
- リスク顕在化コスト（期待値）: ___円/年
合計: ___円/年
```

### Step 2: Value Quantification（価値定量化）

Calculate Total Economic Impact (TEI):

**価値カテゴリ別定量化**:

| カテゴリ | 計算方法 | 例 |
|---------|---------|-----|
| 生産性向上 | 時間削減 × 時給 × 人数 | 2h/日 × 5,000円 × 10人 × 250日 = 25,000,000円 |
| コスト削減 | 削減額（直接） | ライセンス統合: 5,000,000円 |
| 収益増加 | 追加売上 × 利益率 | 売上10%増 × 利益率30% = 30,000,000円 |
| リスク軽減 | 損害額 × 発生確率削減 | 50,000,000円 × 5%削減 = 2,500,000円 |
| 時間価値 | 早期実現による利益 | 3ヶ月早期化 × 月間利益 |

**TEI計算テンプレート**:
```
3年間の総経済効果（TEI）

定量的価値:
- 生産性向上: 75,000,000円
- コスト削減: 15,000,000円
- 収益増加: 90,000,000円
- リスク軽減: 7,500,000円
小計: 187,500,000円

定性的価値（金銭換算困難）:
- 従業員満足度向上
- ブランド価値向上
- 戦略的優位性

割引現在価値（NPV、10%）: 156,200,000円
```

### Step 3: Price Setting（価格設定）

Set price as percentage of value:

**価値取得率ガイドライン**:

| サービスタイプ | 価値取得率 | 理由 |
|--------------|-----------|------|
| コモディティ | 5-10% | 競合多数、差別化困難 |
| 標準サービス | 10-15% | 一定の差別化あり |
| 専門サービス | 15-25% | 高度な専門性 |
| 戦略コンサル | 20-30% | ユニークな価値提供 |
| 独占的サービス | 30-40% | 代替手段なし |

**価格設定計算**:
```
顧客価値（3年NPV）: 156,200,000円
価値取得率: 15%
推奨価格: 23,430,000円

価格帯設定:
- 最低価格: 20,000,000円（価値の12.8%）
- 目標価格: 23,000,000円（価値の14.7%）
- 理想価格: 28,000,000円（価値の17.9%）
```

## Workflow 3: Price Elasticity Analysis（価格弾力性分析）

### Step 1: Method Selection（手法選定）

Select appropriate method using `references/price_elasticity_analysis.md`:

| 手法 | 適用場面 | 精度 | コスト |
|-----|---------|------|-------|
| Van Westendorp | 新製品、価格帯探索 | 中 | 低 |
| Gabor-Granger | 既存製品、価格閾値 | 中〜高 | 中 |
| コンジョイント | 機能×価格トレードオフ | 高 | 高 |
| 回帰分析 | 過去データ活用 | 高 | 低 |

**手法選定フローチャート**:
```
過去の価格×販売データがある？
├─ Yes → 回帰分析
└─ No → 調査ベース
        ├─ 機能トレードオフを分析したい？
        │   ├─ Yes → コンジョイント分析
        │   └─ No → 価格のみ分析
        │           ├─ 新製品/価格帯不明？
        │           │   ├─ Yes → Van Westendorp
        │           │   └─ No → Gabor-Granger
```

```markdown
- サンプルサイズ: 各価格点50名以上（モナディック設計の場合）
```

### Step 3: Elasticity Calculation（弾力性計算）

**価格弾力性の計算式**:

点弾力性:
```
E = (∂Q/Q) / (∂P/P) = (ΔQ/Q) / (ΔP/P)
```

弧弾力性（中点法）:
```
E = [(Q2-Q1)/((Q2+Q1)/2)] / [(P2-P1)/((P2+P1)/2)]
```

**計算例**:
```
価格: 100,000円 → 120,000円（+20%）
販売数: 1,000件 → 850件（-15%）

弾力性 = -15% / 20% = -0.75

解釈: 弾力性 < 1（絶対値）= 非弾力的
→ 価格上昇による収益増が販売減を上回る
```

### Step 4: Demand Curve Construction（需要曲線構築）

Create demand curve from analysis:

```markdown

| 価格 | 予想販売数 | 収益 | 利益 |
|------|-----------|------|------|
| 80,000円 | 1,200件 | 96,000,000円 | 24,000,000円 |
| 100,000円 | 1,000件 | 100,000,000円 | 30,000,000円 |
| 120,000円 | 850件 | 102,000,000円 | 34,000,000円 |
| 140,000円 | 650件 | 91,000,000円 | 32,500,000円 |
| 160,000円 | 450件 | 72,000,000円 | 27,000,000円 |

**最適価格点**: 120,000円（利益最大化）
**収益最大化価格**: 120,000円
```

## Best Practices

1. **Start with Value, Not Cost**: コストではなく顧客価値から価格を考える
2. **Segment Before Pricing**: 一律価格ではなくセグメント別に最適化
3. **Test Before Committing**: 大規模変更前にテスト実施
4. **Update Annually**: 年1回以上の価格見直し（B2B SaaSの94%が実施）
5. **Document Assumptions**: 価格設定の前提条件を明文化
6. **Track Results**: 価格変更の効果を測定・追跡

---

## Common Pitfalls

1. ❌ **Cost-plus Trap**: 戦略サービスにもコストプラスを適用
2. ❌ **Ignoring Competition**: 競合価格を考慮しない
3. ❌ **One-size-fits-all**: 全顧客に同一価格
4. ❌ **No Testing**: テストなしで価格変更を実施
5. ❌ **Over-discounting**: 無規律な値引き
6. ❌ **Price Complexity**: 複雑すぎる価格体系
