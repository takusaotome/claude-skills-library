## 1. TAM/SAM/SOM（市場規模推定）

投資家が最も注目する指標の一つ。「この事業はどこまで大きくなれるか」を示す。

### 定義

- **TAM (Total Addressable Market)**: その製品・サービスが理論上獲得しうる市場全体の規模
- **SAM (Serviceable Addressable Market)**: TAMのうち、自社が実際にアプローチ可能な市場
- **SOM (Serviceable Obtainable Market)**: SAMのうち、現実的に獲得が見込める市場

### 算出方法

**トップダウンアプローチ:**
業界レポートや統計データから全体市場を出発点に絞り込む。
```
TAM = 業界全体の売上高（調査レポート等）
SAM = TAM × 地域限定率 × セグメント限定率
SOM = SAM × 現実的シェア率（通常1〜5%で開始）
```

**ボトムアップアプローチ（推奨）:**
個別の顧客単価と顧客数から積み上げる。
```
SOM = ターゲット顧客数 × 獲得率 × 年間単価
SAM = 対象セグメントの全潜在顧客数 × 年間単価
TAM = 全市場の潜在顧客数 × 年間単価
```

### 注意点
- 投資家はボトムアップの方を信頼する
- TAMが小さすぎると投資対象にならない（一般的に$1B以上が目安）
- SOMは控えめに設定する方が信頼性が上がる

## 2. PEST分析

マクロ環境の変化を4つの視点で分析するフレームワーク。

### 4つの視点

| 要素 | 分析対象 | 具体例 |
|---|---|---|
| **P**olitical（政治） | 法規制、政策、税制 | データ保護法、補助金制度、貿易規制 |
| **E**conomic（経済） | 景気、為替、金利 | GDP成長率、インフレ率、購買力 |
| **S**ocial（社会） | 人口動態、価値観 | 高齢化、リモートワーク普及、環境意識 |
| **T**echnological（技術） | 技術革新、インフラ | AI/ML、5G普及、DX推進 |

### 作成のコツ
- 各要素について「機会」と「脅威」に分類する
- 自社事業に直接影響のある項目に絞る（網羅性より関連性）
- 具体的なデータや事例を引用する
- 将来の変化予測も含める

## 3. 5フォース分析（ポーターの競争戦略）

業界の競争構造と収益性を5つの力で分析する。

### 5つの力

1. **業界内の競争**: 既存企業間の競争の激しさ
2. **新規参入の脅威**: 新たな競合が参入する可能性
3. **代替品の脅威**: 別の手段で同じ課題を解決される可能性
4. **買い手の交渉力**: 顧客がどれだけ価格交渉力を持つか
5. **売り手の交渉力**: サプライヤーがどれだけ価格決定力を持つか

### 評価方法
各力を「弱い・中程度・強い」で評価し、業界全体の魅力度を判定する。
力が弱いほど、事業者にとって有利（利益を上げやすい）。

### 事業計画への活用
- 5つの力が全て強い → 参入を再検討するか、明確な差別化戦略が必要
- 特定の力が弱い → そこを突く戦略を設計
- 自社の参入で力のバランスがどう変わるかも考察する

## 4. SWOT分析

### マトリクス構成

```
             好影響          悪影響
内部要因   Strengths      Weaknesses
外部要因   Opportunities  Threats
```

### クロスSWOT（戦略導出）

単純なSWOTで終わらず、掛け合わせで戦略を導出する:

- **SO戦略（強み×機会）**: 強みを活かして機会を最大化
- **WO戦略（弱み×機会）**: 弱みを克服して機会を活かす
- **ST戦略（強み×脅威）**: 強みで脅威を回避・軽減
- **WT戦略（弱み×脅威）**: 弱みと脅威の最悪シナリオに備える

### 注意点
- 「強み」は顧客にとっての価値で判断する（自社目線にならないように）
- 定性的すぎる分析は避け、可能な限り定量化する
- 競合と比較した相対的な強み・弱みを考える

## 5. ビジネスモデルキャンバス

9つの構成要素で事業モデル全体を1枚の図で可視化する。

### 9つのブロック

1. **顧客セグメント (CS)**: 誰に価値を提供するか
2. **価値提案 (VP)**: どんな価値を提供するか
3. **チャネル (CH)**: どうやって届けるか
4. **顧客との関係 (CR)**: どんな関係を築くか
5. **収益の流れ (R$)**: どうやって収益を得るか
6. **主要リソース (KR)**: 価値を生み出すのに必要なリソース
7. **主要活動 (KA)**: 価値を生み出すための主要な活動
8. **パートナー (KP)**: 外部の協力者
9. **コスト構造 (C$)**: 事業運営にかかるコスト

### 作成順序
VP（価値提案）→ CS（顧客）→ CH（チャネル）→ CR（関係）→ R$（収益）
→ KR（リソース）→ KA（活動）→ KP（パートナー）→ C$（コスト）

## 6. リーンキャンバス

スタートアップ向けにビジネスモデルキャンバスを改良したもの。
不確実性が高い段階で「仮説」を明確にすることが目的。

### 9つのブロック

1. **課題**: ターゲットが抱える上位3つの課題
2. **顧客セグメント**: アーリーアダプター像
3. **独自の価値提案**: 他にない明確な価値（1文で）
4. **ソリューション**: 課題に対する解決策の上位3つ
5. **チャネル**: 顧客にリーチする方法
6. **収益の流れ**: 課金モデル
7. **コスト構造**: 固定費・変動費
8. **主要指標**: 成功を測る3〜5つのKPI
9. **圧倒的な優位性**: 簡単にコピーできない差別化要因

### ビジネスモデルキャンバスとの違い
- 「パートナー」→「課題」に置き換え
- 「主要活動」→「ソリューション」に置き換え
- 「主要リソース」→「主要指標」に置き換え
- 「顧客との関係」→「圧倒的な優位性」に置き換え
- より仮説検証にフォーカスしている

## 7. バリューチェーン分析

事業の主活動と支援活動を分解し、どこで価値が生まれ、どこにコストがかかるかを可視化する。

### 主活動
1. 購買物流（インバウンド）
2. オペレーション（製造・サービス提供）
3. 出荷物流（アウトバウンド）
4. マーケティング・営業
5. サービス（アフターサービス）

### 支援活動
1. 全般管理（経営管理）
2. 人事管理
3. 技術開発
4. 調達

### 活用方法
- 各活動のコストと付加価値を分析
- 競合と比較してどの活動で差別化できるかを特定
- アウトソースすべき活動と内製すべき活動を判断
