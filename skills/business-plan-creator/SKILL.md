# 事業計画書作成スキル (Business Plan Creator)

## 概要

このスキルは、ユーザーの事業アイデアをヒアリングし、体系的な事業計画書を作成するものである。
単なるテンプレート埋めではなく、ユーザーとの対話を通じて事業の本質を深掘りし、
論理的に一貫した計画書を構築する。

## ワークフロー

事業計画の作成は以下の5フェーズで進行する:

```
Phase 1: ヒアリング → Phase 2: 分析 → Phase 3: 戦略設計 → Phase 4: 数値計画 → Phase 5: ドキュメント化
```

### Phase 1: ヒアリング（事業の輪郭を掴む）

ユーザーから以下の情報を聞き出す。一度に全部聞くのではなく、自然な対話の中で段階的に把握する。

**必須ヒアリング項目:**
- 事業の概要（何をやるのか、一言で言うと？）
- ターゲット顧客（誰に売るのか？）
- 提供価値（なぜ顧客はこれを買うのか？）
- 収益モデル（どうやって稼ぐのか？）
- 計画の目的（資金調達、社内承認、自分の整理、など）
- 計画の時間軸（1年、3年、5年）

**任意ヒアリング項目:**
- 既存のリソース（チーム、技術、資金）
- 競合の認識
- 業界経験の有無
- 想定する初期投資額

ユーザーが情報を持っていない項目については、スキップするか、一般的な仮定を置いて進める旨を伝える。

### Phase 2: 分析（市場と競争環境を理解する）

ヒアリング結果を基に、以下の分析フレームワークを適用する。
全てを使う必要はなく、事業の性質に合わせて選択する。

**分析フレームワーク一覧:**

| フレームワーク | 用途 | 使用場面 |
|---|---|---|
| TAM/SAM/SOM | 市場規模の推定 | 新規市場参入時、投資家向け |
| PEST分析 | マクロ環境の把握 | 規制産業、グローバル展開 |
| 5フォース分析 | 業界の競争構造 | 既存市場への参入 |
| SWOT分析 | 自社の立ち位置 | ほぼ全ての事業計画で使用 |
| バリューチェーン | 価値創造プロセス | 製造業、サプライチェーン関連 |
| ビジネスモデルキャンバス | 事業モデル全体の可視化 | スタートアップ、新規事業 |
| リーンキャンバス | 仮説検証型の事業設計 | MVP段階のスタートアップ |

分析結果は事実とデータに基づくこと。推測が入る場合は明示する。
Web検索が利用可能な場合は、市場データや競合情報を積極的に調査する。

各フレームワークの詳細な使い方は `references/frameworks.md` を参照。

### Phase 3: 戦略設計（どう勝つかを決める）

分析結果を踏まえて、以下の戦略要素を設計する:

- **ポジショニング**: 競合との差別化ポイント
- **Go-to-Market戦略**: 市場への参入方法とタイミング
- **マーケティング戦略**: 顧客獲得チャネルと手法
- **組織・チーム計画**: 必要な人材と採用計画
- **マイルストーン**: 主要な達成目標と時期

### Phase 4: 数値計画（財務のリアリティを作る）

財務計画は事業計画の信頼性を左右する最重要セクション。
以下の要素を含める:

**収支計画（P/L）:**
- 売上予測（ユニットエコノミクスベース）
- 原価・変動費
- 固定費（人件費、家賃、システム費用など）
- 営業利益

**資金計画:**
- 初期投資額
- 運転資金
- 資金調達計画（必要な場合）
- ブレークイーブンポイント

**KPI設計:**
- 主要な事業KPIを3〜5個設定
- 測定方法と目標値を明確にする

数値は楽観・標準・悲観の3シナリオで作成することを推奨する。
根拠のない数字は避け、仮定を明記する。

詳細な財務モデリング手法は `references/financial-modeling.md` を参照。

### Phase 5: ドキュメント化（計画書として仕上げる）

最終的なドキュメントの形式はユーザーの目的に応じて選択する:

| 目的 | 推奨形式 | ページ数目安 |
|---|---|---|
| 投資家向けピッチ | PPTX | 10〜15枚 |
| 銀行融資申請 | DOCX | 20〜30ページ |
| 社内稟議・新規事業提案 | DOCX or PPTX | 10〜20ページ |
| 自分の整理用 | Markdown | 制限なし |
| 創業計画書（公的機関） | DOCX（所定フォーマット） | 指定に従う |

ドキュメント作成時は、出力形式に対応する専門スキルが「現在の環境で利用可能か」を確認してから適用する。

1. `skills/` 配下で該当スキルを検索する（例: docx / pptx / xlsx / pdf）。
2. 見つかった場合のみ、そのスキルの `SKILL.md` を参照して出力を作成する。
3. 見つからない場合はMarkdownで納品し、必要に応じて代替形式（PDF化など）を提案する。

## 定型作業の自動化（scripts/）

このスキルには、定型作業を高速化するためのスクリプトを同梱する。

- `scripts/generate_outline.py`:
  目的と業種に応じた事業計画のMarkdownアウトラインを生成する。
  - 例: `python3 skills/business-plan-creator/scripts/generate_outline.py --purpose investor_pitch --industry saas --language ja --company "Acme" --output /tmp/business_plan_outline.md`
- `scripts/generate_financial_scenarios.py`:
  楽観・標準・悲観の3シナリオ財務CSVを生成する。
  - 例: `python3 skills/business-plan-creator/scripts/generate_financial_scenarios.py --base-revenue 12000000 --base-cogs-rate 0.35 --base-opex 450000 --months 12 --output /tmp/financial_scenarios.csv`

## 事業計画書の構成テンプレート

### 標準構成（フル版）

```
1. エグゼクティブサマリー
   - 事業概要（1段落）
   - 解決する課題
   - ソリューション
   - ターゲット市場と規模
   - ビジネスモデル
   - 競争優位性
   - 財務ハイライト
   - 資金需要（該当する場合）

2. 事業の背景と目的
   - 創業の動機・背景
   - ミッション・ビジョン
   - 事業目標（短期・中期・長期）

3. 市場分析
   - 市場規模（TAM/SAM/SOM）
   - 市場トレンド
   - ターゲット顧客像（ペルソナ）
   - 顧客のペインポイント

4. 競合分析
   - 直接競合の分析
   - 間接競合・代替手段
   - 競争優位性・差別化要因
   - 参入障壁

5. 製品・サービス
   - 提供内容の詳細
   - 技術的優位性（該当する場合）
   - 開発ロードマップ
   - 知的財産（該当する場合）

6. ビジネスモデル
   - 収益モデル
   - 価格戦略
   - ユニットエコノミクス
   - スケーラビリティ

7. マーケティング・営業戦略
   - Go-to-Market戦略
   - 顧客獲得チャネル
   - 営業プロセス
   - パートナーシップ戦略

8. オペレーション計画
   - 組織体制
   - 主要メンバー
   - 採用計画
   - 業務フロー

9. 財務計画
   - 前提条件
   - 売上予測（3〜5年）
   - 費用計画
   - 損益計算書
   - キャッシュフロー計画
   - 資金調達計画

10. リスクと対策
    - 主要リスクの特定
    - リスク軽減策
    - コンティンジェンシープラン

11. 実行計画・マイルストーン
    - フェーズ別実行計画
    - 主要マイルストーン
    - KPI一覧

12. 付録
    - 詳細な財務データ
    - 市場調査データ
    - チームの経歴
```

### 簡易構成（社内提案・初期検討用）

```
1. 提案サマリー（1ページ）
2. 課題と機会
3. ソリューション概要
4. 市場・競合の概況
5. 収支概算
6. 実行計画とスケジュール
7. リスクと対策
```

## 出力ルール

- エグゼクティブサマリーは最後に書く（全体が固まってから要約する）
- 数字には必ず根拠・前提条件を付記する
- 競合分析は公平に行い、自社の弱みも正直に記載する
- リスクは具体的に記載し、「リスクはありません」とは書かない
- 専門用語には初出時に簡単な説明を添える
- グラフや図表を積極的に活用する（Excelでの財務モデル作成を含む）

## 言語対応

- ユーザーが日本語で話しかけた場合は日本語で作成
- ユーザーが英語で話しかけた場合は英語で作成
- バイリンガル版を求められた場合は両言語で作成
- 投資家向けの場合は、対象投資家の言語に合わせる
