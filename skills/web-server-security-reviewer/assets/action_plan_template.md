## 4. P0/P1/P2 それぞれの完了判定の標準

### P0 完了判定（インシデント扱い）
- 該当 Finding が `not_observed` 状態に遷移したことの再検証
- 認証情報ローテーション完了の証跡
- 関連システムへの影響なし確認
- ステークホルダー報告完了

### P1 完了判定
- 該当 Finding が `not_observed` または `confirmed_with_mitigation_unverified` → `not_observed` に遷移
- 動作確認テスト（既存機能への影響なし）
- 構成管理ツール（IaC）への反映

### P2 完了判定
- 計画通りの実施完了
- 効果測定（次回レビュー時に Finding 数の減少確認）

## 8. 進捗管理

レビュー完了後、本アクションプランを以下のいずれかで進捗管理:
- Jira / Linear / GitHub Issues 等のチケット化
- 週次レビューミーティングでの進捗確認
- 完了後に再レビュー（横展開 Finding 検出のため）

各項目の状態:
- `pending`: 未着手
- `in_progress`: 着手済
- `mitigation_applied`: 緩和策適用済（動作未検証）
- `verified`: 完了確認済
- `cancelled`: 取り下げ（理由を明記）
