# PMI（Post Merger Integration）計画フレームワーク

## 概要

PMI（Post Merger Integration：買収後統合）は、M&A成功の鍵を握る重要なプロセス。本フレームワークは、クロージング前の準備から長期統合まで、体系的なPMI計画策定を支援する。

---

## 1. PMIフレームワーク全体像

### 1.1 3フェーズ構成

```
┌─────────────────────────────────────────────────────────────────────┐
│  Phase 1: Day 1準備  │  Phase 2: 100日計画  │  Phase 3: 長期統合  │
│    (D-90 ～ D0)      │   (D+1 ～ D+100)    │    (D+100 ～)      │
├─────────────────────────────────────────────────────────────────────┤
│  ・統合計画策定       │  ・クイックウィン     │  ・組織統合完了     │
│  ・IMO設置           │  ・組織統合開始       │  ・システム統合     │
│  ・コミュニケーション計画│  ・シナジー実現開始   │  ・文化融合        │
│  ・Day 1準備         │  ・リテンション       │  ・フルシナジー実現  │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 PMI成功の3要素

| 要素 | 内容 |
|-----|-----|
| **スピード** | 迅速な意思決定と実行 |
| **コミュニケーション** | 全ステークホルダーへの透明性 |
| **人材リテンション** | キーパーソンの確保 |

## 2. Phase 1: Day 1準備（D-90 ～ D0）

```
            [ステアリングコミッティー]

    │        │       │       │        │
```

| 役割 | 責任 |

### 3.3 クイックウィンの特定

**クイックウィン基準:**
- 100日以内に実現可能
- 実現コストが低い
- 効果が可視化しやすい
- リスクが低い

**典型的なクイックウィン:**
| 領域 | 施策 | 効果 |
|-----|-----|-----|
| 調達 | 間接材の統合調達 | コスト削減 |
| IT | 重複ライセンス解約 | コスト削減 |
| 不動産 | 重複オフィス解約 | コスト削減 |
| 営業 | 顧客紹介プログラム開始 | 売上機会 |

**報告サイクル:**

## 6. リスク管理

### 6.1 PMI主要リスク

| リスク | 影響 | 対策 |
|-------|-----|-----|
| 統合遅延 | シナジー遅延、コスト増 | 詳細計画、進捗管理強化 |
| キーパーソン離職 | ナレッジ喪失、業務混乱 | リテンションパッケージ |
| 顧客離反 | 売上減少 | 早期コミュニケーション |
| 文化衝突 | 生産性低下、離職 | 統合後の文化醸成施策 |
| システム障害 | 業務停止、顧客影響 | 段階的統合、テスト強化 |
| 規制対応漏れ | 罰則、レピュテーション | 法務・コンプラ確認 |

### 6.2 リスク管理プロセス

```
[リスク特定] → [影響度・発生確率評価] → [対策立案] → [モニタリング]
                                                    ↑
                                            [定期レビュー]
```

## 7. コミュニケーション計画

### 7.1 ステークホルダー別コミュニケーション

| 対象 | 頻度 | 媒体 | 内容 |
|-----|-----|-----|-----|
| 従業員 | 週次/月次 | 社内ポータル、タウンホール | 統合進捗、FAQ |
| 顧客 | 必要時 | 営業訪問、メール | サービス変更、連絡先 |
| 取引先 | 必要時 | 書面、訪問 | 取引条件変更 |
| 投資家 | 四半期 | IR活動 | 統合効果、業績 |
| 規制当局 | 必要時 | 報告書 | 法定報告 |

### 7.2 コミュニケーション原則

- **透明性**: 隠さず、正直に伝える
- **一貫性**: 全ステークホルダーに同じメッセージ
- **タイムリー**: 変更・決定を速やかに伝達
- **双方向**: フィードバックを受け付ける
