# Vendor Evaluation Criteria

## Overview

This document provides a comprehensive framework for evaluating vendor proposals in software development procurement. The framework ensures objective, consistent, and defensible vendor selection decisions.

## Standard Evaluation Dimensions

### 1. Price (Weight: 25-35%)

**Objective:** Assess cost competitiveness and value for money.

**Scoring Criteria:**

| Score | Description |
|-------|-------------|
| 100 | Lowest price among compliant bids |
| 80-99 | Within 10% of lowest price |
| 60-79 | Within 20% of lowest price |
| 40-59 | Within 30% of lowest price |
| 0-39 | More than 30% above lowest price |

**Formula (Relative Scoring):**
```
Score = (Lowest Price / Vendor Price) × 100
```

**Evaluation Points:**
- Total project cost
- Unit pricing transparency
- Hidden cost identification
- Payment terms flexibility
- Volume discount potential

### 2. Technical Capability (Weight: 20-30%)

**Objective:** Assess vendor's ability to deliver technically.

**Scoring Criteria:**

| Score | Description |
|-------|-------------|
| 90-100 | Exceeds requirements with innovative approach |
| 70-89 | Fully meets all requirements |
| 50-69 | Meets most requirements with minor gaps |
| 30-49 | Significant gaps requiring mitigation |
| 0-29 | Major gaps, high risk |

**Evaluation Points:**
- Technical solution architecture
- Technology stack appropriateness
- Scalability approach
- Security implementation
- Integration methodology
- Development practices (CI/CD, testing)
- Code quality standards

### 3. Experience and Track Record (Weight: 15-25%)

**Objective:** Assess vendor's relevant experience and reliability.

**Scoring Criteria:**

| Score | Description |
|-------|-------------|
| 90-100 | 5+ similar projects, excellent references |
| 70-89 | 3-4 similar projects, good references |
| 50-69 | 1-2 similar projects, acceptable references |
| 30-49 | Related experience only, limited references |
| 0-29 | No relevant experience |

**Evaluation Points:**
- Number of similar projects
- Client references (verifiable)
- Industry expertise
- Project success rate
- Client retention rate
- Case study quality

### 4. Project Management (Weight: 10-20%)

**Objective:** Assess vendor's ability to manage delivery effectively.

**Scoring Criteria:**

| Score | Description |
|-------|-------------|
| 90-100 | Comprehensive PM approach with proven tools |
| 70-89 | Solid PM methodology with clear processes |
| 50-69 | Standard PM approach, some gaps |
| 30-49 | Basic PM only, concerns about control |
| 0-29 | Inadequate PM capability |

**Evaluation Points:**
- Project management methodology
- Communication plan
- Risk management approach
- Change management process
- Quality assurance methods
- Status reporting frequency
- Escalation procedures

### 5. Timeline and Delivery (Weight: 10-15%)

**Objective:** Assess realistic delivery capability.

**Scoring Criteria:**

| Score | Description |
|-------|-------------|
| 90-100 | Ahead of required timeline with buffer |
| 70-89 | Meets required timeline |
| 50-69 | Slightly exceeds timeline (<10%) |
| 30-49 | Exceeds timeline (10-20%) |
| 0-29 | Significantly exceeds timeline (>20%) |

**Evaluation Points:**
- Overall timeline realism
- Milestone definition
- Resource availability
- Dependency management
- Buffer allocation
- Parallel work streams
- Critical path clarity

### 6. Support and Maintenance (Weight: 5-10%)

**Objective:** Assess post-delivery support capability.

**Scoring Criteria:**

| Score | Description |
|-------|-------------|
| 90-100 | Comprehensive SLA with 24/7 support option |
| 70-89 | Standard SLA with business hours support |
| 50-69 | Basic support with limitations |
| 30-49 | Minimal support commitment |
| 0-29 | No clear support offering |

**Evaluation Points:**
- Support hours and availability
- Response time SLAs
- Support team structure
- Knowledge transfer plan
- Documentation quality
- Warranty period
- Maintenance pricing

## Evaluation Process

### Step 1: Pre-Evaluation Setup

1. Confirm evaluation criteria and weights
2. Identify evaluation team (minimum 3 evaluators)
3. Prepare evaluation forms
4. Schedule evaluation sessions

### Step 2: Individual Evaluation

1. Each evaluator reviews proposals independently
2. Score each dimension using criteria above
3. Document rationale for each score
4. Identify questions for clarification

### Step 3: Consensus Building

1. Compile individual scores
2. Discuss significant score variations (>15 points)
3. Reach consensus on final scores
4. Document any unresolved disagreements

### Step 4: Risk Assessment

1. Identify vendor-specific risks
2. Assess impact and likelihood
3. Factor risk into final evaluation
4. Document risk mitigation requirements

### Step 5: Final Recommendation

1. Calculate weighted total scores
2. Rank vendors by total score
3. Consider qualitative factors
4. Prepare selection recommendation

## Qualitative Factors

Beyond numerical scoring, consider these qualitative factors:

### Cultural Fit
- Communication style compatibility
- Working hours alignment
- Geographic proximity
- Language proficiency

### Strategic Value
- Long-term partnership potential
- Innovation capability
- Market position
- Growth trajectory

### Risk Profile
- Financial stability
- Key person dependency
- Subcontractor usage
- Insurance coverage

## Red Flags

Watch for these warning signs during evaluation:

### Pricing Red Flags
- Significantly lower than other bids (>30%)
- Vague pricing structure
- Excessive change order provisions
- Front-loaded payment terms

### Technical Red Flags
- Overly complex solutions
- Unproven technology stack
- Missing security considerations
- No scalability plan

### Experience Red Flags
- Unable to provide references
- Inconsistent project history
- High staff turnover
- Recent legal issues

### PM Red Flags
- Unclear resource commitment
- No named project manager
- Missing milestone definitions
- Excessive optimism

## Documentation Requirements

### Evaluation File Must Include

1. **Evaluation summary** with final scores
2. **Individual evaluator worksheets**
3. **Score calculation documentation**
4. **Risk assessment findings**
5. **Reference check results**
6. **Clarification Q&A records**
7. **Decision rationale memo**

### Retention Requirements

- Maintain evaluation records for minimum 3 years
- Store in secure, accessible location
- Ensure audit trail completeness

## Special Considerations

### Small Vendor Evaluation

- May lack extensive references
- Focus on team expertise over company size
- Consider growth potential
- Assess financial stability more carefully

### Offshore/Nearshore Vendors

- Add communication assessment
- Consider timezone impact
- Evaluate IP protection measures
- Include travel cost estimates

### Consortium/Partnership Bids

- Clarify lead partner responsibilities
- Assess integration between partners
- Verify joint delivery experience
- Understand liability structure

## Scoring Adjustments

### Bonus Points (up to +10%)
- Innovative approach beyond requirements
- Exceptional references with site visits
- Significant cost reduction proposals
- Value-added services included

### Penalty Points (up to -10%)
- Incomplete proposal submission
- Late submission without approval
- Non-compliance with mandatory requirements
- Inconsistencies in proposal
