# Data Governance（データガバナンス）

## 概要

データガバナンスは、DAMA-DMBOKの中心に位置する知識領域です。組織のデータ資産に対する権限と統制を行使し、データの価値を最大化しながらリスクを管理します。

---

## データガバナンスの定義

> データ資産の管理に対する権限と統制の行使。
> データに関する意思決定権と説明責任を定義するシステム。

---

## ガバナンスの目的

1. **価値の創出**: データを戦略的資産として活用
2. **リスクの管理**: コンプライアンス、セキュリティ
3. **コストの最適化**: 重複排除、効率化
4. **品質の確保**: 信頼できるデータの維持

---

### 典型的な組織構造

```
┌─────────────────────────────────────────────────────────┐
│                  経営層/取締役会                         │
│                 （最終意思決定）                         │

                         │
┌────────────────────────▼────────────────────────────────┐
│              データガバナンス委員会                      │

└────────────────────────┬────────────────────────────────┘
                         │
         ┌───────────────┼───────────────┐
         │               │               │
         ▼               ▼               ▼
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│ Data Owner  │  │ Data Owner  │  │ Data Owner  │
│ (顧客領域)  │  │ (製品領域)  │  │ (財務領域)  │
└──────┬──────┘  └──────┬──────┘  └──────┬──────┘
       │               │               │
       ▼               ▼               ▼
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│Data Steward │  │Data Steward │  │Data Steward │
│ (日次管理)  │  │ (日次管理)  │  │ (日次管理)  │
└─────────────┘  └─────────────┘  └─────────────┘
```

### 主要な役割

| 役割 | 責任 | 典型的な担当者 |
|------|------|---------------|
| **CDO** | 全社データ戦略、ガバナンス統括 | 役員レベル |
| **Data Owner** | 領域のデータに対する最終責任 | 部門長、マネージャー |
| **Data Steward** | 日常的なデータ管理、品質維持 | 業務担当者 |
| **Data Custodian** | 技術的なデータ管理 | IT担当者 |

## ガバナンス活動

### 1. 戦略策定

- データビジョンの定義
- 戦略目標の設定
- ロードマップの作成
- 投資計画の策定

### 2. ポリシー・標準策定

**主要なポリシー:**
- データ分類ポリシー
- データ品質ポリシー
- データアクセスポリシー
- データ保持ポリシー
- データセキュリティポリシー

### 3. 組織・役割定義

- ガバナンス組織の設計
- 役割と責任の定義（RACI）
- 意思決定プロセスの定義
- エスカレーションパスの定義

### 4. 監視・測定

**主要なKPI:**

| カテゴリ | KPI例 |
|---------|-------|
| 品質 | データ品質スコア、エラー率 |
| 利用 | データカタログ利用率 |
| コンプライアンス | ポリシー遵守率 |
| 価値 | データ活用による売上貢献 |

### 5. コンプライアンス管理

- 規制要件の特定
- コンプライアンス評価
- 是正措置の実施
- 監査対応

## ガバナンス導入アプローチ

### 段階的アプローチ

```
Phase 1: 基盤構築（3-6ヶ月）
├── ガバナンス組織の設立
├── 基本ポリシーの策定
├── パイロット領域の選定
└── クイックウィンの実現

Phase 2: 展開（6-12ヶ月）
├── 標準・ガイドラインの整備
├── データオーナー・スチュワード任命
├── ツール導入
└── 対象領域の拡大

Phase 3: 定着（12-24ヶ月）
├── 全社展開
├── 測定・改善サイクル確立
├── 教育・啓発
└── 文化醸成
```

### 成功要因

1. **経営層のコミットメント**: トップダウンの支持
2. **明確な価値提案**: ビジネス価値の可視化
3. **適切なスコープ**: 小さく始めて拡大
4. **人材・組織**: 適切な役割と権限
5. **継続的改善**: PDCAサイクル

### 失敗パターン

- ✗ IT主導でビジネス不在
- ✗ 一度に全てを完璧にしようとする
- ✗ ツール導入だけで終わる
- ✗ ポリシーを作っても運用されない
- ✗ 測定・モニタリングがない

## ガバナンスとコンプライアンス

### 主要な規制

| 規制 | 対象 | 主な要件 |
|------|------|----------|
| **GDPR** | EU個人データ | 同意、削除権、ポータビリティ |
| **CCPA** | カリフォルニア州 | 開示、オプトアウト |
| **個人情報保護法** | 日本国内 | 利用目的、第三者提供 |
| **SOX** | 上場企業 | 財務データの正確性 |
| **HIPAA** | 医療情報 | プライバシー保護 |

### ガバナンスとの関係

```
規制要件 ──────────▶ ポリシー ──────────▶ 標準・手順 ──────────▶ 実装

  └───▶ 監査・評価 ◀───┴────▶ モニタリング ◀─┴────▶ 記録・証跡 ◀──┘
```
