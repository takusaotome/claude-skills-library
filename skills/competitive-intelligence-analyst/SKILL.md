# Competitive Intelligence Analyst

## 概要

競合分析と市場調査を体系的に支援する専門スキルです。営業支援、戦略立案、製品開発に必要な競合情報の収集・分析・活用を包括的にカバーします。

**主な用途:**
- 市場調査・競合ベンチマーク
- 営業支援（バトルカード作成）
- 商談分析（Win/Loss分析）
- 戦略立案支援

**出力形式:**
- 競合プロファイル（日本語/英語）
- Win/Loss分析レポート（日本語/英語）
- バトルカード（日本語/英語）
- 価格比較表
- 技術比較マトリクス
- 市場ランドスケープマップ

## コアワークフロー

1. **競合特定・優先順位付け** - 分析対象となる競合企業の特定と優先度設定
2. **競合プロファイル作成** - 個別競合企業の詳細分析
3. **Win/Loss分析** - 商談の勝敗要因分析
4. **バトルカード作成** - 営業向け競合対策資料
5. **市場ランドスケープ分析** - 市場全体の競合マップ作成
6. **継続的モニタリング計画** - 競合動向の継続追跡

## ワークフロー詳細

### ワークフロー 1: 競合特定・優先順位付け

**目的:** 分析リソースを効果的に配分するため、競合企業を特定し優先順位を付ける

**ステップ:**

1. **市場定義の確認**
   - ユーザーに事業領域・製品カテゴリを確認
   - ターゲット顧客セグメントを明確化
   - 地理的範囲を特定

2. **競合候補のリストアップ**
   ```
   質問例:
   - 現在認識している主な競合企業は？
   - 顧客から名前が挙がる他社は？
   - 最近参入してきた新規プレイヤーは？
   - 代替ソリューション（間接競合）は？
   ```

3. **競合分類**
   | カテゴリ | 定義 | 例 |
   |---------|------|-----|
   | 直接競合 | 同一製品・同一市場 | 同業他社 |
   | 間接競合 | 異なる製品・同一ニーズ | 代替ソリューション |
   | 潜在競合 | 参入可能性のある企業 | 隣接市場プレイヤー |

4. **優先順位付け**
   - **Tier 1（最優先）**: 頻繁に競合、市場シェア大
   - **Tier 2（重要）**: 時々競合、成長中
   - **Tier 3（監視）**: 潜在的脅威、新規参入者

5. **分析計画の策定**
   - Tier別の分析深度を決定
   - リソース配分を計画
   - 更新頻度を設定

---

### ワークフロー 2: 競合プロファイル作成

**目的:** 個別競合企業の包括的な分析資料を作成する

**ステップ:**

1. **情報収集**
   - `references/intelligence_sources.md` を参照し、情報源を特定
   - 公開情報（10K/10Q、プレスリリース、LinkedIn等）を収集
   - 業界レポート、アナリストレポートを活用

2. **基本情報の整理**
   ```
   収集項目:
   - 設立年、本社所在地
   - 従業員数、売上規模
   - 資金調達状況（スタートアップの場合）
   - 主要顧客セグメント
   - 経営陣プロフィール
   ```

3. **製品・サービス分析**
   - 製品ラインナップの整理
   - 価格帯の調査
   - 機能比較表の作成

4. **SWOT分析の実施**
   - `references/competitive_analysis_frameworks.md` を参照
   - 強み・弱み・機会・脅威を分析
   - 当社との相対評価を実施

5. **Go-to-Market戦略の分析**
   - 販売チャネル
   - マーケティング手法
   - パートナーエコシステム

6. **プロファイルの出力**
   - `assets/competitor_profile_template_ja.md` または `_en.md` を使用
   - 全セクションを記入
   - 情報の鮮度を明記

### ワークフロー 3: Win/Loss分析

**目的:** 商談の勝敗要因を分析し、営業戦略を改善する

**ステップ:**

1. **分析対象の特定**
   ```
   質問例:
   - 分析期間は？（例: 過去6ヶ月）
   - 対象商談数は？
   - 勝ち/負けの定義は？
   - 競合別に分けるか？
   ```

2. **データ収集**
   - `references/win_loss_methodology.md` を参照
   - CRM/SFAからのデータ抽出
   - 営業担当者へのインタビュー
   - 可能であれば顧客インタビュー

3. **勝因分析**
   | カテゴリ | 要因例 |
   |---------|--------|
   | 製品 | 機能優位、使いやすさ、統合性 |
   | 価格 | 価格競争力、TCO優位 |
   | 関係性 | 既存関係、信頼性 |
   | 営業 | 提案力、対応速度 |
   | ブランド | 認知度、評判 |

4. **敗因分析**
   - 同カテゴリで敗因を分類
   - 頻出パターンを特定
   - 競合別の敗因傾向を分析

5. **アクション導出**
   - 勝因の強化策
   - 敗因の改善策
   - 競合別の対策

6. **レポート出力**
   - `assets/win_loss_analysis_template_ja.md` または `_en.md` を使用
   - 定量データとインサイトを統合
   - 推奨アクションを明記

---

### ワークフロー 4: バトルカード作成

**目的:** 営業が商談で使える実践的な競合対策資料を作成する

**ステップ:**

1. **対象競合の確認**
   - バトルカード作成対象の競合を特定
   - 競合プロファイルが存在するか確認
   - 最新情報が反映されているか確認

2. **差別化ポイントの整理**
   - `references/battlecard_best_practices.md` を参照
   - 当社の強みを3-5点に絞る
   - 競合の弱みを3-5点に絞る
   - 顧客視点での価値を明確化

3. **反論処理の準備**
   ```
   よくある反論:
   Q: 「競合Aの方が安いのでは？」
   A: 「初期費用は競合Aが安いですが、
      TCOで比較すると当社の方が20%低くなります。
      具体的には...」
   ```

4. **価格比較の作成**
   - `references/pricing_analysis_guide.md` を参照
   - 価格モデルの違いを説明
   - TCO比較を準備
   - 値引き交渉への対応策

5. **勝ち事例の収集**
   - この競合に勝った事例を2-3件
   - 顧客名（可能であれば）
   - 勝因のポイント

6. **バトルカード出力**
   - `assets/battlecard_template_ja.md` または `_en.md` を使用
   - 30秒で要点が伝わる構成
   - 更新日を明記

---

### ワークフロー 5: 市場ランドスケープ分析

**目的:** 市場全体の競合状況を俯瞰的に把握する

**ステップ:**

1. **市場定義**
   ```
   確認事項:
   - 市場の範囲（製品カテゴリ、地域）
   - 市場規模と成長率
   - 主要セグメント
   ```

2. **競合マップの作成**
   - `references/competitive_analysis_frameworks.md` を参照
   - 2x2マトリクスの軸を決定
     - 例: 価格 vs 機能、市場シェア vs 成長率
   - 各競合をプロット

3. **市場シェア分析**
   | 企業 | 推定シェア | トレンド |
   |-----|-----------|---------|
   | 当社 | 15% | ↑ |
   | 競合A | 25% | → |
   | 競合B | 20% | ↓ |

4. **セグメント別分析**
   - セグメントごとの競合状況
   - 各社の強いセグメント
   - 空白地帯の特定

5. **トレンド・動向**
   - 市場トレンド
   - 技術トレンド
   - 規制動向

6. **機会と脅威**
   - 市場機会の特定
   - 脅威となる動向
   - 戦略的含意

7. **レポート出力**
   - `assets/market_landscape_template.md` を使用
   - 視覚的なマップを含める
   - アクショナブルな洞察を提供

---

### ワークフロー 6: 継続的モニタリング計画

**目的:** 競合動向を継続的に追跡し、タイムリーに対応する

**ステップ:**

1. **モニタリング対象の設定**
   - Tier 1競合: 週次
   - Tier 2競合: 月次
   - Tier 3競合: 四半期

2. **情報源の設定**
   - `references/intelligence_sources.md` を参照
   - Googleアラート設定
   - SNS/LinkedInフォロー
   - 業界ニュースサイト

3. **トラッキング項目**
   ```
   監視項目:
   - 製品アップデート・新製品発表
   - 価格変更
   - 人事異動（特に経営陣）
   - 資金調達・M&A
   - パートナーシップ
   - 顧客獲得・喪失
   - マーケティングキャンペーン
   ```

4. **アラート基準**
   - 即時エスカレーション: 価格変更、大型案件
   - 週次レポート: 製品アップデート、人事
   - 月次レビュー: トレンド、戦略変更

5. **レビュープロセス**
   - 週次: 営業チームとの共有
   - 月次: 競合情報アップデート会議
   - 四半期: プロファイル・バトルカード更新

## ベストプラクティス

1. **ファクトベース**: 推測ではなく、検証可能な情報に基づく
2. **定期更新**: 情報の鮮度を保つ（最低四半期ごと）
3. **アクショナブル**: 分析結果は具体的なアクションに繋げる
4. **顧客視点**: 競合比較は顧客にとっての価値で評価
5. **倫理的収集**: 合法的かつ倫理的な情報収集に限定
6. **バイアス排除**: 自社有利なバイアスを排除した客観分析
7. **クロスファンクショナル**: 営業、マーケ、製品チームと連携

---

## よくある間違い

- **情報過多**: 使いきれない情報を収集しすぎる
- **更新不足**: 古い情報のまま意思決定する
- **主観的評価**: 客観的データなしに評価する
- **営業との乖離**: 現場で使われない資料を作成する
- **法的リスク**: 不適切な情報収集手法を使用する
- **一回限り**: 継続的な更新プロセスを確立しない
