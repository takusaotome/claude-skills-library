# Competitive Intelligence Sources

競合情報を収集するための情報源と手法ガイド。

---

## 情報収集の原則

### 倫理的ガイドライン

**DO（すべきこと）:**
- 公開情報のみを収集
- 合法的な手段で情報を入手
- 情報源を明記
- 収集した情報を適切に管理

**DON'T（避けるべきこと）:**
- 虚偽の身分を使った情報収集
- 機密情報の不正取得
- 競合社員への不適切な接触
- 著作権・利用規約の違反

---

## 公開情報源

### 1. 企業公式情報

| 情報源 | 入手できる情報 | 更新頻度 |
|--------|--------------|---------|
| **企業Webサイト** | 製品情報、価格、事例、経営陣 | 随時 |
| **プレスリリース** | 新製品、パートナーシップ、業績 | 随時 |
| **IRページ（上場企業）** | 財務情報、戦略、リスク要因 | 四半期 |
| **採用ページ** | 組織拡大方向、技術スタック | 随時 |

### 2. 財務情報（上場企業）

**米国企業:**
- **10-K**: 年次報告書（事業概要、リスク、財務）
- **10-Q**: 四半期報告書
- **8-K**: 重要イベントの報告
- **入手先**: SEC EDGAR (https://www.sec.gov/edgar)

**日本企業:**
- **有価証券報告書**: 年次報告書
- **四半期報告書**: 四半期財務情報
- **入手先**: EDINET (https://disclosure.edinet-fsa.go.jp)

**読み方のポイント:**
```
注目セクション:
- Business Description: 事業概要、競合認識
- Risk Factors: 認識しているリスク
- Management Discussion: 経営陣の見解
- Revenue Breakdown: 収益構造
```

## 情報収集プロセス

### 日次ルーティン（15分）

```
□ Googleアラートの確認
□ 競合SNSのチェック
□ 業界ニュースのスキャン
□ 重要情報のフラグ付け
```

### 週次ルーティン（1時間）

```
□ 競合Webサイトの更新確認
□ 価格ページの確認
□ 新しい顧客事例の確認
□ 採用ページの確認
□ レビューサイトの新着確認
```

### 月次ルーティン（2時間）

```
□ プレスリリースのまとめ
□ 財務情報の更新（上場企業）
□ LinkedIn経由の組織変更確認
□ 競合プロファイルの更新
□ チームへの共有
```

### 四半期ルーティン（4時間）

```
□ 業界レポートの確認
□ 包括的な競合分析
□ バトルカードの更新
□ 戦略レビュー会議
```

## 情報の信頼性評価

### 信頼性スコアリング

| レベル | 情報源 | 信頼度 |
|--------|--------|--------|
| **A** | 公式発表、財務報告、規制提出書類 | 高 |
| **B** | 信頼できるメディア報道、アナリストレポート | 中〜高 |
| **C** | 業界関係者からの情報、顧客からの情報 | 中 |
| **D** | SNS投稿、匿名レビュー | 低〜中 |
| **E** | 噂、未確認情報 | 低 |

### 検証のポイント

```
✓ 複数ソースでの確認
✓ 情報の鮮度（いつの情報か）
✓ 情報源の利害関係
✓ 具体性（数字、日付、名前）
✓ 文脈の理解
```

---

## 情報収集のアンチパターン

| アンチパターン | 問題点 | 対策 |
|--------------|--------|------|
| **情報過多** | 分析が追いつかない | 優先度を設定 |
| **古い情報への依存** | 誤った判断 | 定期更新を仕組み化 |
| **偏った情報源** | バイアスの発生 | 複数ソースを確認 |
| **噂への反応** | 信頼性の問題 | 必ず検証 |
| **倫理違反** | 法的リスク | ガイドライン遵守 |

## 情報共有のベストプラクティス

### 社内共有の仕組み

**1. リアルタイム共有（Slack/Teams）**
```
#competitive-intel チャンネル
- 速報性のある情報
- 価格変更、製品発表
- 競合の大型案件獲得
```

**2. 定期レポート**
```
週次ダイジェスト:
- 今週の主要ニュース
- 競合の動き
- 要注意事項

月次レポート:
- 競合動向のまとめ
- トレンド分析
- 戦略への示唆
```

**3. ナレッジベース**
```
構造:
- 競合プロファイル（競合別）
- バトルカード（競合別）
- 業界レポート
- 分析テンプレート
```

### セキュリティ考慮事項

- 機密情報のアクセス制限
- 外部共有の禁止
- 情報の分類とラベリング
- 定期的な棚卸し
