# Severity Criteria (Document Review)

文書レビュー指摘事項の重大度判定基準。統合フェーズで各ペルソナの Impact 記述をこの基準に照合して最終判定する。

---

## 重大度定義

### Critical（致命的）

**定義**: 即座に修正が必要。このまま進めると失敗リスクが大きい。

**基準**:
- 主張に対する根拠が全くない
- 重大な論理誤謬（循環論法、因果の錯誤）
- 重要な前提条件が未検証
- セキュリティ上の重大な見落とし
- テスト不可能な受入基準で次工程に進めない

**アクション**: 次工程進行前に必ず修正

**例**:

| 問題 | 理由 |
|------|------|
| 結論に証拠がない | 判断根拠なしでプロジェクトが進む |
| 循環論法で正当化 | 論理が成立していない |
| 重要な前提が未検証 | 前提崩壊時にプロジェクト失敗 |
| 認証方式が未定義 | セキュリティリスク |
| 受入基準が曖昧で検証不可 | 品質保証が不可能 |

### Major（重大）

**定義**: 修正が強く推奨される。実害を招く可能性がある。

**基準**:
- 根拠が不十分で重要判断に使用されている
- 曖昧さが実務上の問題を招く可能性
- テスト困難な要件が複数存在
- 運用時の障害対応が定義されていない
- ステークホルダー間の認識齟齬を生む可能性

**アクション**: 次工程開始前に修正を推奨

**例**:

| 問題 | 理由 |
|------|------|
| 「〜と思われる」が重要判断に | 根拠不十分なまま判断 |
| エッジケースが未定義 | テスト工程で手戻り |
| 障害時の復旧手順なし | 運用リスク |
| 整合性のない記述 | 実装時の混乱 |

### Minor（軽微）

**定義**: 改善が望ましいが緊急ではない。影響は限定的。

**基準**:
- 曖昧だが影響範囲が限定的
- 表現の改善が望ましい
- 軽微な不整合

**アクション**: 可能であれば修正

### Info（参考）

**定義**: 参考情報、改善提案。修正は任意。

**基準**:
- より良い表現の提案
- 参考文献の追加推奨
- ベストプラクティスの紹介

**アクション**: 著者の判断に委ねる

---

## 判定フローチャート

```
指摘事項
├── 根拠が全くない or 論理誤謬 → Critical
├── 根拠不十分 + 重要判断に関わる → Major
├── 曖昧だが影響限定的 → Minor
└── 改善提案レベル → Info
```

## マージ/進行判定

| 判定 | 条件 |
|------|------|
| **進行可** | Critical: 0件 |
| **条件付き進行** | Critical: 0件, Major: 軽微 |
| **進行不可** | Critical: 1件以上 |
