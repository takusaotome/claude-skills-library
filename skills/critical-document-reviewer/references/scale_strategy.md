# Scale Strategy (Document Review)

大規模ドキュメントのレビュー時に、効率的に重要セクションを特定し
重点レビューを行うための戦略。

---

## 発動条件

以下のいずれかに該当する場合にこの戦略を適用:
- 文字数 **30,000字超**
- 見出し (`#`) 数 **30超**
- 表・コードブロック数 **20超**

## Phase A: 優先セクション抽出

### 優先度の高いセクション（Stage 1 対象）

以下のキーワードを含む見出しを持つセクションを優先:

| 優先度 | 見出しキーワード |
|--------|--------------|
| 最高 | 結論、まとめ、判断、決定、方針 |
| 高 | リスク、制約、前提条件、注意事項 |
| 高 | 要件、仕様、受入基準 |
| 中 | 設計、アーキテクチャ、構成 |
| 中 | 数値、データ、メトリクス、KPI |

### 優先度判定方法

1. 全見出しを Grep で抽出
2. 上記キーワードとのマッチングで優先度を付与
3. 上位セクションを Stage 1 の対象とする

## Phase B: 2段階レビュー

### Stage 1: 優先セクション重点レビュー

優先度の高いセクションに対して:
- 選定ペルソナ（3〜6ペルソナ）を並列実行
- Token Budget Strategy に従いリファレンスを選択

### Stage 2: 残りの主張/根拠チェック

優先セクション以外に対して:
- 単一パスで主張と根拠の対応のみチェック
- Critical レベルの問題のみ検出を目指す
- 対象: 根拠なき主張、重大な論理矛盾、事実誤認

## スケール報告

レビューレポートに以下を含める:
- 文書の規模情報（文字数、見出し数）
- Stage 1 でレビューしたセクション
- Stage 2 でレビューしたセクション
- 未レビューのセクションがある場合はその旨を明記
