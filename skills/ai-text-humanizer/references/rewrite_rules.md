# Rewrite Rules by Pattern

パターン別の具体的な変換ルール。検出されたパターンに対応する変換手順とBefore/After例。

## 変換の優先順位

高い影響度のパターンから順に適用する:

1. **非コミット姿勢** → 最も読者が違和感を感じるパターン
2. **マニュアル的構成** → 構造レベルの変更で全体の印象が変わる
3. **視覚的マーカー** → 機械的に除去しやすい
4. **単調なリズム** → 文末・文長のバリエーション追加
5. **抽象語の濫用** → 具体化には文脈理解が必要
6. **定型メタファー** → 削除または具体表現への置換

---

## Rule 1: 視覚的マーカーの除去

文書種別で扱いを切り替える:

- メール本文/チャット: Markdown装飾と構造を除去してプレーンテキスト化
- ブログ記事: 見出し・箇条書きなどの構造は保持
- 構造化文書（見積書、提案書、報告書、仕様書、設計書）: 見出し、表、箇条書きは保持

### Transform Rules

| Before | After | Rule |
|--------|-------|------|
| `**重要なポイント**` | 重要なポイント | 過剰な太字を削減、強調は文脈で表現 |
| `## セクション名` | `## セクション名`（ブログ/構造化文書）/ 改行区切り（メール） | 文書種別で保持か変換を選択 |
| `—` (エムダッシュ) | 「。」や接続助詞 | 自然な接続に置換 |
| `／` | 「や」「あるいは」 | 日本語の選択表現に |
| `（補足説明）` | 本文に溶け込ませる | 括弧を減らす |
| `- 項目1\n- 項目2` | 箇条書き維持（ブログ/構造化文書）/ 文章化（メール） | 文書種別で処理を切替 |

### Example

**Before:**
> **リモートワーク**には、以下のメリットがあります：
> - 通勤時間の削減
> - ワークライフバランスの向上
> - コスト削減（オフィス賃料／交通費）

**After:**
> リモートワークのメリットは、通勤がなくなること、仕事とプライベートの切り分けがしやすくなること、あとはオフィス代や交通費が浮くこと。この3つが大きい。

## Rule 2: リズムの変化

### Transform Rules

| Before | After | Rule |
|--------|-------|------|
| 「〜です。〜です。〜です。」 | 「〜だ。〜である。〜になる。」 | 文末を混ぜる |
| 「また、〜。さらに、〜。」 | 接続詞を削除し直接つなぐ | 接続詞を減らす |
| 均一な文長（40-50字） | 短文(15字)と長文(60字)を混ぜる | 文長に変化をつける |
| 「〜だけでなく〜も」 | 「〜だ。それに〜もある」 | not A but Bを分解 |

### Example

**Before:**
> デジタル化は業務効率を向上させます。また、コスト削減にも貢献します。さらに、顧客体験の改善にもつながります。しかし、導入には課題もあります。

**After:**
> デジタル化すると業務が速くなる。コストも下がる。顧客の体験も変わる。ただ、導入はそう簡単じゃない。

## Rule 3: マニュアル的構成の解体

### Transform Rules

| Before | After | Rule |
|--------|-------|------|
| 「本稿では〜について解説します」 | 削除、いきなり本題へ | 構成宣言を除去 |
| 長い背景説明 → 本題 | 本題 → 必要な背景 | 結論ファーストに |
| 「ステップ1:〜 ステップ2:〜」 | 自然な流れの説明 | ステップ表記を排除 |
| 「まとめると〜が重要です」 | 具体的な一言で締める | 薄い結論を具体化 |
| 均等なセクション | 重要度に応じて傾斜 | 密度に変化をつける |

### Example

**Before:**
> 近年、AI技術の進化は目覚ましく、多くの産業に影響を与えています。本稿では、AI活用の現状、主要な技術トレンド、企業における導入事例、そして今後の展望について、包括的に解説します。
>
> まず、AI活用の現状について見ていきましょう。

**After:**
> ChatGPTが出てから2年、企業のAI導入は加速した。ただ、実際に成果を出せている企業は2割程度だろう。残りの8割は「とりあえず入れてみた」で止まっている。

## Rule 4: 非コミット姿勢の除去

### Transform Rules

| Before | After | Rule |
|--------|-------|------|
| 「〜かもしれません」 | 「〜だ」「〜だろう」 | 断定に変換 |
| 「〜の可能性があります」 | 「〜になる」 | 可能性を確信に |
| 「一概には言えませんが」 | 削除 | 免責表現を除去 |
| 「一方で〜という見方も」 | 自分の立場を明示 | 強制中立を排除 |
| 「〜傾向にあります」 | 「〜だ」 | 断定回避を除去 |
| 「メリットとデメリット」 | 自分の評価を述べる | 両論併記を止める |

### Example

**Before:**
> この手法は効果的である可能性がありますが、状況によって異なる場合もあり、一概には言えません。メリットとしては〜、デメリットとしては〜が挙げられます。総合的に判断することが重要ではないでしょうか。

**After:**
> この手法は効果がある。ただし条件がある。チームの規模が10人以下で、イテレーションサイクルが2週間以内の場合だ。それ以外では別のやり方を勧める。

## Rule 5: 抽象語の具体化

### Transform Rules

| Before | After | Rule |
|--------|-------|------|
| 「包括的なアプローチ」 | 具体的な手法名を挙げる | 抽象→具体 |
| 「効果的な施策」 | 具体的な施策内容 | 何が効果的か明示 |
| 「飛躍的な向上」 | 「30%の改善」等の数値 | 定量化 |
| 「戦略的な判断」 | 何をどう判断したか | 判断内容を明示 |
| 「最適な〜」 | 最適の基準を明示 | 基準を具体化 |
| 「フレームワーク」 | 枠組みの中身を説明 | カタカナを開く |

### Example

**Before:**
> 包括的なデータ分析アプローチにより、戦略的な意思決定を支援し、持続可能な成長を実現するための最適なソリューションを提供します。

**After:**
> 売上データ、顧客行動ログ、NPSの3つを突き合わせて、どの施策が利益に効いているかを月次で出す。経営会議の前に必ずこのレポートを見てもらう。

## Rule 6: 定型メタファーの処理

### Transform Rules

| Before | After | Rule |
|--------|-------|------|
| 「羅針盤となる」 | 「方向を示す」「判断基準になる」 | 具体的な機能を記述 |
| 「DNAに刻む」 | 「根付いている」「当たり前になっている」 | 日常語に |
| 「車の両輪」 | 「両方必要だ」「片方だけではダメ」 | 直接的に言い換え |
| 「エンジンとなる」 | 「推進力になる」「原動力だ」 | 平易な表現に |
| 「青写真を描く」 | 「計画を立てる」「構想する」 | 一般語に |
| 「架け橋となる」 | 「つなぐ」「仲介する」 | 動詞に変換 |

### Principle

定型メタファーは2つの処理方法がある:

1. **削除**: メタファーがなくても意味が通じる場合は単純に削除
2. **具体化**: メタファーが指す内容を具体的に書き換える

メタファー自体が悪いのではなく、LLM特有の使い古されたメタファーが問題。独自のメタファーや具体的な比喩は残してよい。

### Example

**Before:**
> ビジョンは組織の羅針盤であり、この青写真に基づいて戦略というエンジンを回すことで、イノベーションという果実を手にすることができるのです。

**After:**
> ビジョンがないとチームは迷う。何を優先するか、何を捨てるかが決められない。戦略はそのビジョンを行動計画に落とし込んだものだ。
