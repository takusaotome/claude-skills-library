## Purpose

This matrix defines the single source of truth (SSOT) for every evidence item used in quality gate evaluations. For each item, it specifies whether the source is automated or manual, who owns the accuracy of the data, how frequently it is updated, and what rule applies when values in different documents diverge.

## Auto vs Manual Classification Rules

### Auto (Automated Source)

- Generated by a CI pipeline, tool, or system without human editing
- Stored as a versioned artifact with immutable content
- Identified by CI run number, commit hash, or artifact digest
- **Cannot be manually overridden** unless the auto source is demonstrably broken (requires exception)

### Manual (Human-Authored Source)

- Created or completed by a human (sign-off forms, UAT notes, exception records)
- Stored in a durable, addressable location (wiki page, document repository)
- Must include author name, date, and version
- Acceptable only for items that inherently require human judgment

### Classification Decision Guide

| Question | If Yes | If No |
|----------|--------|-------|
| Can a CI job produce this metric? | Auto | Consider manual |
| Does this require human judgment to evaluate? | Manual | Likely auto |
| Is the metric a count or measurement from a tool? | Auto | -- |
| Is the metric an approval or acceptance decision? | Manual | -- |
| Can two people produce different values for the same input? | Likely manual (standardize if possible) | Likely auto |

## Reconciliation Quick Reference

### When Auto and Manual Sources Disagree

1. **Auto source takes precedence** (default rule)
2. If auto source is suspected to be wrong:
   - File an issue against the CI configuration
   - Document the discrepancy in the exception register
   - Use the corrected manual value WITH explicit annotation referencing the issue
3. Fix the auto source before the next gate review

### When Two Auto Sources Disagree

1. Check timestamps: use the more recent value if within the reconciliation window
2. Check scope: ensure both sources measure the same thing
3. If genuine conflict: investigate root cause before using either value

### Reconciliation Window

| Document Type | Max Staleness |
|---------------|---------------|
| Gate review evidence | Same CI run as release candidate (0 staleness) |
| Project dashboard | 24 hours |
| Status reports | Current sprint |
