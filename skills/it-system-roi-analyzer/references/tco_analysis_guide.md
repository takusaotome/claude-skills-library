# TCO分析ガイド（Total Cost of Ownership）

## Overview

TCO（総所有コスト）は、ITシステムの導入から運用・廃棄までのライフサイクル全体にかかるコストを包括的に把握するための分析手法。

---

## TCOの構成要素

### コスト構造

```
TCO（5年間）
├── 1. 初期投資（Initial Investment）
│   ├── システム開発・導入費
│   ├── ハードウェア購入費
│   ├── ソフトウェアライセンス（初期）
│   ├── インフラ構築費
│   ├── データ移行費
│   ├── トレーニング費
│   └── コンティンジェンシー
├── 2. 運用コスト（Operating Cost）
│   ├── 人件費（運用・保守）
│   ├── クラウド/ホスティング費
│   ├── ライセンス費（年間）
│   ├── 保守サポート費
│   ├── ネットワーク・回線費
│   └── 施設費（電力・空調）
└── 3. 廃棄・移行コスト（Exit Cost）
    ├── データ移行費
    ├── システム廃棄費
    └── 契約終了関連費
```

---

## 初期投資（Initial Investment）

### システム開発・導入費

| 項目 | 内容 | 算出方法 |
|------|------|----------|
| 要件定義 | 業務分析、要件整理 | 工数 × 単価 |
| 設計 | 基本設計、詳細設計 | 工数 × 単価 |
| 開発 | プログラミング、単体テスト | 工数 × 単価 |
| テスト | 結合・総合テスト、UAT | 工数 × 単価 |
| 導入 | 本番環境構築、移行 | 工数 × 単価 |

**目安（パッケージ導入の場合）:**
- ライセンス費の1.5-3倍が導入費用

### ハードウェア購入費

| 項目 | 内容 | 耐用年数 |
|------|------|----------|
| サーバー | 物理サーバー、仮想基盤 | 5年 |
| ストレージ | SAN、NAS | 5年 |
| ネットワーク機器 | スイッチ、ルーター、FW | 5-7年 |
| クライアント端末 | PC、タブレット | 3-4年 |

### ソフトウェアライセンス（初期）

| ライセンス形態 | 特徴 | TCO考慮事項 |
|---------------|------|------------|
| 永続ライセンス | 一括購入 | 初期費用大、保守費別途 |
| サブスクリプション | 年間/月額課金 | 初期費用小、運用費で計上 |
| ユーザー課金 | ユーザー数×単価 | スケール時のコスト注意 |
| CPU/コア課金 | CPU数×単価 | 仮想化時の課金注意 |

### その他初期費用

| 項目 | 目安 |
|------|------|
| データ移行 | 総開発費の5-15% |
| トレーニング | ユーザー数 × 1-3万円 |
| コンサルティング | 総プロジェクト費の10-20% |
| コンティンジェンシー | 総投資の10-15% |

---

## 運用コスト（Operating Cost）

### 人件費

| 役割 | 内容 | 算出方法 |
|------|------|----------|
| システム管理者 | サーバー・NW運用 | FTE × 年間人件費 |
| アプリケーション管理者 | アプリ運用・設定変更 | FTE × 年間人件費 |
| ヘルプデスク | ユーザーサポート | FTE × 年間人件費 |
| セキュリティ担当 | セキュリティ監視・対応 | FTE × 年間人件費 |

**人件費単価目安（日本）:**
- 一般運用担当: 600-800万円/年
- 専門エンジニア: 800-1,200万円/年
- 外部委託: 150-300万円/人月

### クラウド/ホスティング費

| サービス | 課金モデル | 注意事項 |
|---------|-----------|----------|
| IaaS | 従量課金 | 使用量予測、リザーブド活用 |
| PaaS | 従量課金 | トランザクション数予測 |
| SaaS | ユーザー課金 | ユーザー数変動考慮 |

**クラウドコスト見積もり例:**
```
AWS/Azure/GCP月額費用:
├── コンピューティング: XX万円
├── ストレージ: XX万円
├── データ転送: XX万円
├── データベース: XX万円
└── その他サービス: XX万円
──────────────────────────
合計: XX万円/月 → XX万円/年
```

### ライセンス費（年間）

| 形態 | 算出方法 |
|------|----------|
| 保守サポート費 | 永続ライセンスの15-22%/年 |
| サブスクリプション | 年間契約額 |
| 従量課金 | 使用量 × 単価 |

### 保守サポート費

| レベル | 内容 | 目安 |
|--------|------|------|
| 基本保守 | 障害対応、パッチ提供 | 開発費の15%/年 |
| 標準保守 | 基本+問い合わせ対応 | 開発費の18%/年 |
| プレミアム保守 | 標準+24/365対応 | 開発費の22%/年 |

---

## 廃棄・移行コスト（Exit Cost）

次期システムへの移行時に発生するコスト：

| 項目 | 内容 |
|------|------|
| データ移行 | 既存データの抽出・変換・投入 |
| 並行稼働 | 新旧システム並行期間のコスト |
| 契約解除 | 早期解約ペナルティ、残存リース |
| 廃棄処理 | ハードウェア処分、データ消去 |

## オンプレ vs クラウド TCO比較

### 比較項目

| 項目 | オンプレミス | クラウド |
|------|-------------|---------|
| 初期投資 | 大（HW購入） | 小（従量課金） |
| 運用コスト | 固定費中心 | 変動費中心 |
| スケーラビリティ | 追加投資必要 | 柔軟に拡張可能 |
| 廃棄コスト | HW処分費用 | 契約終了のみ |
| 人件費 | インフラ運用人員必要 | 削減可能 |
| 陳腐化リスク | 高い | 低い（常に最新） |

### 損益分岐点

一般的な傾向：
- **3年未満**: クラウドが有利
- **3-5年**: ワークロード次第
- **5年超**: オンプレが有利（ただし陳腐化リスク）

## TCO削減のポイント

### 初期投資削減
1. クラウドファースト検討
2. パッケージ/SaaS活用
3. 段階的導入
4. 複数ベンダー比較

### 運用コスト削減
1. 自動化（Infrastructure as Code）
2. マネージドサービス活用
3. リザーブドインスタンス
4. 適正サイジング

### 隠れたコストの可視化
1. シャドーIT の洗い出し
2. 非効率な運用プロセスの特定
3. 未使用リソースの特定
4. ベンダー契約の見直し
