# IT ROI分析方法論

## Overview

ITシステム導入・刷新プロジェクトのROI（投資対効果）を分析するための体系的な方法論。本ガイドはIT投資に特化した分析手法を提供する。

---

## ROI分析の目的

1. **投資判断**: プロジェクトに投資する価値があるかを定量的に評価
2. **優先順位付け**: 複数のIT投資候補から優先順位を決定
3. **ステークホルダー説得**: 経営層、財務部門への説得材料
4. **プロジェクト正当化**: プロジェクト承認のための根拠
5. **効果測定**: プロジェクト完了後の効果検証

---

## IT投資特有の考慮事項

### 技術的リスク

| リスク | 影響 | 対策 |
|--------|------|------|
| 技術陳腐化 | 投資価値の低下 | 段階的投資、クラウド活用 |
| ベンダーロックイン | 将来コスト増 | 標準技術採用、マルチベンダー |
| スケーラビリティ不足 | 追加投資必要 | 拡張性設計、クラウドネイティブ |
| セキュリティ脆弱性 | 損害・信用低下 | セキュリティ投資込みで計算 |

### IT投資のライフサイクル

```
導入期（Year 0-1）→ 安定期（Year 2-3）→ 成熟期（Year 4-5）→ 衰退期（Year 6+）
```

- **導入期**: 効果発現は限定的（50-70%程度）
- **安定期**: フル効果発現（100%）
- **成熟期**: 効果維持、若干の追加効果
- **衰退期**: 技術陳腐化、次期投資検討

## 分析フレームワーク

### 1. As-Is分析（現状分析）

**収集すべき情報:**

#### 直接コスト
```
現行IT関連コスト:
├── ハードウェア費用
│   ├── サーバー（減価償却/リース）
│   ├── ストレージ
│   ├── ネットワーク機器
│   └── クライアント端末
├── ソフトウェア費用
│   ├── ライセンス費（年間/永続）
│   ├── 保守サポート費
│   └── カスタマイズ費
├── 人件費
│   ├── IT運用担当
│   ├── 開発・保守担当
│   └── ヘルプデスク
├── 外部委託費
│   ├── ベンダー保守
│   ├── 運用アウトソーシング
│   └── コンサルティング
└── 施設費
    ├── データセンター
    ├── 電力・空調
    └── 回線費用
```

#### 間接コスト（隠れたコスト）
```
業務影響コスト:
├── ダウンタイム損失
│   └── 停止時間 × 時間当たり売上・生産損失
├── 生産性損失
│   ├── システム待ち時間
│   ├── 手作業・二重入力
│   └── データ不整合対応
├── 機会損失
│   ├── 新機能追加の遅延
│   ├── スケーラビリティ制約
│   └── 市場対応の遅れ
└── リスクコスト
    ├── セキュリティインシデント
    ├── コンプライアンス違反
    └── 事業継続リスク
```

### 2. To-Be定義（将来像）

**新システムの期待効果:**

| カテゴリ | 効果項目 | 定量化方法 |
|---------|---------|------------|
| コスト削減 | IT運用費削減 | 旧コスト - 新コスト |
| コスト削減 | 人件費削減 | 削減FTE × 年間人件費 |
| 生産性向上 | 処理時間短縮 | 短縮時間 × 時間単価 × 処理回数 |
| 生産性向上 | スループット向上 | 追加処理能力 × 単価 |
| リスク低減 | ダウンタイム削減 | 削減時間 × 時間損失 |
| リスク低減 | セキュリティ強化 | 想定損失 × 確率低減 |
| 売上貢献 | 新規事業創出 | 新規売上 × 利益率 |
| 売上貢献 | 顧客満足度向上 | 解約率低減 × 顧客売上 |

## 財務指標の算出

### ROI（Return on Investment）

**定義**: 投資に対してどれだけの利益が得られるかを示す指標

**計算式:**
```
ROI (%) = ((5年間総効果 - 5年間総投資) / 5年間総投資) × 100
```

**IT投資における注意点:**
- 効果発現の遅延を考慮（1年目は50%想定等）
- 隠れたコストを含める
- 機会費用も考慮

**判断基準:**
| ROI | 評価 | アクション |
|-----|------|-----------|
| 100%以上 | 非常に良い | 優先投資対象 |
| 50-100% | 良い | 投資推奨 |
| 20-50% | 許容範囲 | 戦略的判断で投資可 |
| 0-20% | 要検討 | 他の選択肢と比較 |
| 0%未満 | 投資非推奨 | 再検討または中止 |

---

### NPV（Net Present Value）

**定義**: 将来のキャッシュフローを現在価値に割り引いた合計

**計算式:**
```
NPV = Σ (CFt / (1 + r)^t) - 初期投資

Where:
- CFt = t年目のキャッシュフロー
- r = 割引率（WACC等）
- t = 年数
```

**IT投資での割引率設定:**
| プロジェクトタイプ | リスク | 割引率 |
|------------------|--------|--------|
| インフラ更新 | 低 | 8-10% |
| パッケージ導入 | 中 | 10-12% |
| 新規開発 | 中-高 | 12-15% |
| 先端技術採用 | 高 | 15-20% |

---

### IRR（Internal Rate of Return）

**定義**: NPVをゼロにする割引率

**計算方法**: Newton-Raphson法による反復計算

**IT投資でのIRR判断:**
- IRR > WACC（資本コスト）: 投資価値あり
- IRR > ハードルレート（通常15%）: 積極的に投資
- IRR < WACC: 投資価値なし

---

### 投資回収期間（Payback Period）

**計算式:**
```
回収期間 = 累積キャッシュフローが初期投資を超える年数
```

**IT投資の回収期間目安:**
| システム種別 | 目標回収期間 |
|-------------|-------------|
| インフラ更新 | 3-5年 |
| 業務アプリ | 2-4年 |
| 顧客向けシステム | 1-3年 |
| 戦略的投資 | 3-5年（戦略的価値考慮） |

## Do-Nothingシナリオ

現状維持の場合のコスト・リスクを明示：

### Do-Nothingコスト

```
5年間の現状維持コスト:
├── 継続的な運用コスト増（年率3-5%増）
├── 老朽化によるメンテナンスコスト増
├── セキュリティパッチ対応コスト
├── 機会損失（新機能追加不可）
└── リスク増大（セキュリティ、可用性）
```

### 比較表

| 項目 | Do-Nothing（5年） | 新システム導入（5年） |
|------|------------------|---------------------|
| 総コスト | XX万円 | YY万円 |
| リスク | 増大 | 低減 |
| 生産性 | 低下 | 向上 |
| 競争力 | 低下 | 維持/向上 |

## ベストプラクティス

### 1. 保守的に見積もる
- **効果**: 確実に達成可能な80%水準で見積もり
- **コスト**: コンティンジェンシー15-20%を含める
- **期間**: 効果発現は段階的（1年目50%、2年目以降100%）

### 2. 複数シナリオで分析
- 楽観シナリオ（効果+20%、コスト-10%）
- 標準シナリオ（計画値）
- 悲観シナリオ（効果-20%、コスト+10%）
- 最悪シナリオ（効果-30%、コスト+20%）

### 3. 定量化困難な効果の扱い
- 戦略的価値として別記
- 可能な限り金額換算を試みる
- 財務指標計算には含めず、追加価値として提示

### 4. ステークホルダー別の説明
| ステークホルダー | 関心事項 | 説明のポイント |
|-----------------|---------|---------------|
| 経営層 | 戦略的価値、ROI | エグゼクティブサマリー重視 |
| 財務部門 | NPV、IRR、キャッシュフロー | 財務指標の詳細 |
| IT部門 | 技術的実現可能性、運用負荷 | TCO、運用コスト |
| 事業部門 | 業務改善効果 | 定量効果の内訳 |
