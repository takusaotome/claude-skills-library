# Meeting Minutes Quality Checklist

Use this checklist to validate meeting minutes before distribution.

## Pre-Distribution Checklist

### Header Information
- [ ] Meeting date and time clearly stated
- [ ] Meeting type/purpose identified
- [ ] Location or video platform specified
- [ ] Full attendee list with names and roles
- [ ] Absentees noted (if relevant)
- [ ] Meeting facilitator/chair identified
- [ ] Note taker identified

### Agenda Coverage
- [ ] All agenda items addressed in minutes
- [ ] Additional topics marked as "Added Items"
- [ ] Skipped items noted with reason
- [ ] Time allocations reasonable per item

### Discussion Summaries
- [ ] Key points captured for each topic
- [ ] Different viewpoints noted where relevant
- [ ] Questions raised are documented
- [ ] Concerns or risks identified are listed
- [ ] No confidential information inappropriately disclosed

### Decisions Documentation
- [ ] Each decision clearly stated
- [ ] Context for decision provided
- [ ] Alternatives considered noted (where applicable)
- [ ] Rationale for decision documented
- [ ] Vote count or consensus noted (if applicable)
- [ ] Dissenting opinions recorded (if significant)

### Action Items
- [ ] Every action item has a single owner
- [ ] Every action item has a specific deadline (date, not relative)
- [ ] Action descriptions are clear and specific
- [ ] Actions link to relevant decision or discussion
- [ ] No orphan actions (actions without clear source)
- [ ] Status tracking method defined (optional)

### Next Steps
- [ ] Follow-up meeting scheduled (if needed)
- [ ] Key milestones or deadlines noted
- [ ] Dependencies between actions identified
- [ ] Escalation path defined for blockers

### Language Quality
- [ ] Professional and neutral tone
- [ ] No ambiguous pronouns
- [ ] Specific terms over vague language
- [ ] Consistent terminology throughout
- [ ] Acronyms defined on first use
- [ ] Easy to scan for key information

### Formatting
- [ ] Consistent heading structure
- [ ] Proper use of lists and tables
- [ ] Clear visual separation between sections
- [ ] Page numbers (for longer documents)
- [ ] Version number or date in footer

## Action Item Quality Criteria

### Must Have (All Required)
1. **Owner**: Single named individual (not team/group)
2. **Deadline**: Specific date (YYYY-MM-DD format preferred)
3. **Description**: Clear, actionable task description

### Should Have
4. **Context**: Reference to source discussion or decision
5. **Priority**: If competing with other items
6. **Dependencies**: If blocked by other actions

### Format Template
```
- [ ] @Owner Name: Clear action description
      Due: YYYY-MM-DD
      Priority: High/Medium/Low
      Context: Discussion item #N / Decision #M
```

## Decision Documentation Criteria

### Must Have
1. **Decision statement**: What was decided
2. **Context**: Why decision was needed

### Should Have
3. **Alternatives**: Options that were considered
4. **Rationale**: Why this option was selected
5. **Impact**: Who or what is affected

### Format Template
```
### Decision #N: [Short Decision Title]

**Decision**: [Clear statement of what was decided]

**Context**: [Why this decision was needed]

**Alternatives Considered**:
1. Option A — [Brief description]
2. Option B — [Brief description]

**Rationale**: [Why this option was chosen]

**Impact**: [Who/what is affected, follow-up required]
```

## Common Quality Issues

### Completeness Issues
| Issue | Detection | Fix |
|-------|-----------|-----|
| Missing attendees | Compare with invite list | Add missing names |
| Skipped agenda item | Compare with agenda | Add discussion or note as skipped |
| No decisions section | Search for implicit decisions | Extract and document |
| Missing next steps | Check for follow-up plans | Add if discussed |

### Action Item Issues
| Issue | Detection | Fix |
|-------|-----------|-----|
| No owner | Search for team/group assignment | Assign individual |
| Vague deadline | Search "soon", "later", "ASAP" | Replace with date |
| Unclear description | Search verbs without objects | Add specific details |
| Orphan action | No link to discussion | Add context reference |

### Decision Issues
| Issue | Detection | Fix |
|-------|-----------|-----|
| Implicit decision | Look for "agreed", "decided" | Make explicit |
| Missing rationale | Decision without "because" | Add reasoning |
| No alternatives | Complex decisions need options | Document if discussed |
| Vague impact | Missing who/what affected | Add impact statement |

### Clarity Issues
| Issue | Detection | Fix |
|-------|-----------|-----|
| Ambiguous pronoun | "they", "it", "this" unclear | Replace with noun |
| Vague language | "improve", "look into" | Add specifics |
| Jargon | Technical terms | Define or simplify |
| Passive voice | "should be done" | Add owner/actor |

## Review Workflow

1. **First Pass**: Structure check
   - All required sections present?
   - Proper formatting?

2. **Second Pass**: Content check
   - Agenda items covered?
   - Decisions documented?
   - Action items complete?

3. **Third Pass**: Quality check
   - Clear language?
   - Consistent terminology?
   - No ambiguity?

4. **Final Pass**: Source verification (if available)
   - Matches hearing sheet/agenda?
   - Attendees accurate?
   - Key points captured?

## Red Flags (Block Distribution)

The following issues should block distribution until resolved:

1. **Missing critical decision** documented in source materials
2. **Action item without owner** for critical task
3. **Incorrect date/time** in meeting header
4. **Wrong attendees** listed (especially decision-makers)
5. **Confidential information** inappropriately included
6. **Factually incorrect** statements about decisions or discussions
