# Gap Analysis Methodology for ISO Implementation

## Purpose

This document provides a systematic methodology for conducting gap analysis against ISO management system standards. The gap analysis serves as the foundation for implementation planning and resource allocation.

## Gap Analysis Process

### Phase 1: Preparation

#### 1.1 Define Scope
- Identify organizational units included
- Specify geographic locations
- List products/services in scope
- Document any planned exclusions

#### 1.2 Identify Standard Requirements
- Obtain current version of target ISO standard
- Map all mandatory requirements (SHALL statements)
- Identify recommended practices (SHOULD statements)
- Note informative guidance (annexes, notes)

#### 1.3 Gather Documentation
Collect existing documentation:
- Policies and manuals
- Procedures and work instructions
- Forms and templates
- Records and evidence
- Organization charts
- Process maps
- Previous audit reports

#### 1.4 Schedule Interviews
- Management representatives
- Process owners
- Key staff members
- Subject matter experts

### Phase 2: Assessment

#### 2.1 Document Review
For each clause, assess:
- Does documentation exist?
- Is it current and approved?
- Does it meet requirements?
- Is it accessible to relevant personnel?

#### 2.2 Interview Execution
Standard questions for each process:
1. "Describe how this process works in practice."
2. "What records do you maintain?"
3. "How do you know when this process is effective?"
4. "What training have you received?"
5. "Show me an example from last month."

#### 2.3 Evidence Sampling
Select samples to verify:
- Consistency between documentation and practice
- Record completeness and accuracy
- Training effectiveness
- Process output quality

### Phase 3: Scoring

#### 3.1 Compliance Level Assessment

| Score | Definition | Evidence Criteria |
|-------|------------|-------------------|
| **5** | Fully Compliant | Process documented, consistently implemented, records complete, measurable results, continuous improvement evident |
| **4** | Mostly Compliant | Process documented and implemented, minor gaps in records or consistency, improvement activities exist |
| **3** | Partially Compliant | Process exists but not fully documented OR documented but inconsistently followed, limited records |
| **2** | Minimally Compliant | Awareness exists, some ad-hoc practices, no formal documentation, minimal records |
| **1** | Not Compliant | No process, no documentation, no evidence of compliance |
| **0** | Not Applicable | Justified exclusion (document rationale) |

#### 3.2 Risk-Based Prioritization

Assess each gap by:

**Impact (1-5)**:
- 5: Critical to certification / legal requirement
- 4: Major process dependency
- 3: Moderate effect on compliance
- 2: Minor effect
- 1: Administrative only

**Effort (1-5)**:
- 5: >6 months, significant resources
- 4: 3-6 months, dedicated resources
- 3: 1-3 months, moderate effort
- 2: 2-4 weeks, limited effort
- 1: <2 weeks, minimal effort

**Priority Matrix**:
```
        High Impact (4-5)
             │
  Quick Win  │  Strategic
  (Do First) │  (Plan Carefully)
 ───────────┼───────────
  Consider   │  Low Priority
  (If Time)  │  (Do Last)
             │
        Low Impact (1-3)

   Low Effort ──────────── High Effort
     (1-3)                   (4-5)
```

### Phase 4: Analysis

#### 4.1 Calculate Maturity Score

**Overall Maturity** = Sum of Scores / Number of Applicable Clauses

**Interpretation**:
- 4.5-5.0: Ready for certification
- 3.5-4.4: Minor improvements needed
- 2.5-3.4: Significant work required
- 1.5-2.4: Major implementation needed
- 1.0-1.4: Starting from scratch

#### 4.2 Identify Patterns

Look for systemic issues:
- Documentation gaps across multiple clauses
- Training deficiencies
- Record-keeping weaknesses
- Management engagement issues
- Resource constraints

#### 4.3 Benchmark Against Timeline

Typical effort by starting maturity:

| Starting Maturity | Typical Timeline to Certification |
|-------------------|-----------------------------------|
| 4.0+ | 3-6 months |
| 3.0-3.9 | 6-9 months |
| 2.0-2.9 | 9-12 months |
| <2.0 | 12-18 months |

### Phase 5: Reporting

#### 5.1 Executive Summary
- Overall maturity score
- Number of gaps by severity
- Estimated timeline
- Resource requirements
- Key recommendations

#### 5.2 Detailed Findings
For each clause:
- Requirement description
- Current state
- Compliance score
- Gap description
- Evidence reviewed
- Recommended actions
- Priority level
- Estimated effort

#### 5.3 Action Plan
Prioritized list of actions:
- Action description
- Owner
- Due date
- Resources needed
- Dependencies
- Success criteria

## Quality Assurance

### Calibration
- Use multiple assessors and compare results
- Reference previous audit findings
- Validate with process owners

### Objectivity
- Assess based on evidence, not intent
- Document all findings with specific examples
- Distinguish between "not seen" and "does not exist"

### Completeness
- Cover all applicable requirements
- Include organizational context
- Consider all locations in scope

## Output Artifacts

1. **Gap Analysis Workbook**: Excel/CSV with all clause assessments
2. **Gap Analysis Report**: Executive-friendly summary document
3. **Action Plan**: Prioritized implementation roadmap
4. **Evidence Log**: References to reviewed documents and records
