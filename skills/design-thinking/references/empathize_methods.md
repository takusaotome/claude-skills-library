# Empathize Methods（共感フェーズの手法）

## 概要

共感フェーズの目的は、ユーザーの**表面的なニーズ**ではなく、**深層のニーズ（インサイト）**を発見することです。

```
表面的ニーズ: 「速い車が欲しい」
       ↓ 深掘り
深層のニーズ: 「移動時間を無駄にしたくない」
       ↓ さらに深掘り
インサイト: 「自分の時間をコントロールしたい」
```

---

## 1. インタビュー

### 1.1 インタビューの種類

| 種類 | 深さ | 人数 | 時間 |
|------|------|------|------|
| **構造化インタビュー** | 浅 | 多 | 15-30分 |
| **半構造化インタビュー** | 中 | 中 | 30-60分 |
| **深層インタビュー** | 深 | 少 | 60-90分 |
| **コンテキストインタビュー** | 深 | 少 | 2-3時間 |

### 1.2 質問の種類

**オープンエンド質問（推奨）:**
- 「〇〇についてどのように感じていますか？」
- 「最近、〇〇で印象に残った出来事を教えてください」
- 「理想的にはどうなっていると良いですか？」

**深掘り質問:**
- 「なぜそう思うのですか？」（5 Whys）
- 「具体的に教えてください」
- 「そのとき、どう感じましたか？」
- 「他に方法はありましたか？」

**避けるべき質問:**
- ✗ 誘導質問：「〇〇は便利だと思いませんか？」
- ✗ Yes/No質問：「〇〇を使っていますか？」
- ✗ 仮定の質問：「もし〇〇があったら使いますか？」

### 1.3 インタビューのコツ

```
【聞く姿勢】
・8割聞く、2割話す
・沈黙を恐れない（考える時間を与える）
・メモより目を見る（録音に頼る）

【深掘りのタイミング】
・感情が見えたとき
・曖昧な表現のとき
・矛盾を感じたとき
・興味深いエピソードのとき

【記録のポイント】
・発言をそのまま記録（解釈しない）
・非言語情報も記録（表情、声のトーン）
・印象に残った瞬間をマーク
```

## 2. 観察（Observation）

### 2.1 観察の種類

| 種類 | 説明 | 適用場面 |
|------|------|----------|
| **参与観察** | 一緒に活動しながら観察 | 深い理解が必要なとき |
| **非参与観察** | 離れた場所から観察 | 自然な行動を見たいとき |
| **シャドーイング** | 1日密着して行動を追う | 日常全体を理解したいとき |
| **フライオンザウォール** | 存在を消して観察 | 影響を与えたくないとき |

### 2.2 観察のポイント

```
【見るべきもの】
・行動と発言の矛盾
・工夫・カスタマイズ
・困っている瞬間
・感情の変化
・環境・文脈

【記録フォーマット】
時刻 | 行動 | 発言 | 感情 | 気づき
-----|------|------|------|-------
10:05| ...  | ...  | 😐   | ...
```

### 2.3 AEIOU フレームワーク

観察を整理するためのフレームワーク:

| 要素 | 観察ポイント |
|------|-------------|
| **A**ctivities | 何をしているか？ゴールは？ |
| **E**nvironments | どんな環境か？物理的特徴は？ |
| **I**nteractions | 誰と、何と、どう関わっているか？ |
| **O**bjects | どんな物を使っているか？ |
| **U**sers | どんな人か？役割、価値観は？ |

## 3. エンパシーマップ

### 3.1 構成要素

```
┌─────────────────────────────────────────────────────┐
│                    [ユーザー名]                      │
│                    [簡単な説明]                      │
├────────────────────────┬────────────────────────────┤
│         SAYS           │          THINKS            │
│       （言うこと）       │        （考えること）       │
│                        │                            │
│ ・インタビューでの発言   │ ・本当は何を考えているか    │
│ ・よく使う言葉          │ ・言わないけど思っていること │
│ ・口癖                  │ ・優先順位                  │
│                        │                            │
├────────────────────────┼────────────────────────────┤
│         DOES           │          FEELS             │
│        （行動）         │         （感情）            │
│                        │                            │
│ ・観察された行動        │ ・喜び、満足               │
│ ・習慣的な行動          │ ・不安、恐れ               │
│ ・回避行動              │ ・フラストレーション        │
│                        │                            │
├────────────────────────┴────────────────────────────┤
│                       PAINS                          │
│                   （ペイン・課題）                    │
│                                                      │
│ ・障壁、ブロッカー                                    │
│ ・フラストレーションの原因                            │
│ ・恐れ、不安                                         │
├──────────────────────────────────────────────────────┤
│                       GAINS                          │
│                   （ゲイン・欲求）                    │
│                                                      │
│ ・達成したいこと                                     │
│ ・成功の定義                                         │
│ ・期待を超える体験                                   │
└──────────────────────────────────────────────────────┘
```

### 3.2 作成のコツ

1. **SAYSから始める**: 事実から始めると作りやすい
2. **矛盾を探す**: SAYSとDOESの矛盾がインサイトのヒント
3. **チームで作る**: 複数人の視点を統合
4. **仮説を明記**: THINKSとFEELSは仮説であることを意識

## 4. 5 Whys（なぜなぜ分析）

### 4.1 基本的な進め方

```
問題: 「ユーザーがアプリを使い続けない」

Why 1: なぜ使い続けないのか？
  → 「通知が多すぎてうっとうしいから」

Why 2: なぜ通知を減らさないのか？
  → 「設定方法がわからないから」

Why 3: なぜ設定方法がわからないのか？
  → 「設定画面が見つけにくいから」

Why 4: なぜ設定画面が見つけにくいのか？
  → 「メニュー階層が深いから」

Why 5: なぜメニュー階層が深いのか？
  → 「機能追加のたびに項目が増えたから」

→ 根本原因: 情報設計の見直しが必要
```

### 4.2 注意点

- 責任追及にならないよう注意
- 複数の「なぜ」が出ることもある（分岐OK）
- 5回にこだわらない（3回で十分な場合も）

## 5. Extreme Users（極端なユーザー）

### 5.1 なぜ極端なユーザーに注目するか

- 一般ユーザーでは見えにくいニーズが顕在化
- 革新的なソリューションのヒントが得られる
- 少数のインタビューで深いインサイトが得られる

### 5.2 極端なユーザーの例

| カテゴリ | 極端なユーザー例 |
|---------|-----------------|
| 使用頻度 | ヘビーユーザー / 全く使わない人 |
| 習熟度 | エキスパート / 完全初心者 |
| 環境 | 最先端環境 / リソース制約環境 |
| 身体 | 障害のある方 / アスリート |
| 状況 | 極度のストレス下 / 余裕のある状況 |

## 6. 成果物のまとめ

| 成果物 | 目的 | 形式 |
|--------|------|------|
| インタビューメモ | 生の声を記録 | テキスト/音声 |
| 観察ノート | 行動を記録 | フィールドノート |
| エンパシーマップ | 洞察を統合 | キャンバス |
| インサイトステートメント | 発見を言語化 | 1文の洞察 |
| 写真・動画 | 状況を記録 | メディアファイル |
