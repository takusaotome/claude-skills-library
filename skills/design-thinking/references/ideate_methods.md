# Ideate Methods（アイデア発想フェーズの手法）

## 概要

アイデア発想フェーズでは、**発散（量を出す）**→**収束（選ぶ）**の2段階で進めます。

```
┌─────────────────────────────────────────────────────────┐
│                    Ideateフェーズ                       │
│                                                         │
│     発散（Diverge）          収束（Converge）           │
│                                                         │
│         ╲     ╱                 ╲     ╱                │
│          ╲   ╱                   ╲   ╱                 │
│           ╲ ╱                     ╲ ╱                  │
│            ╳   →→→→→→→→→→→→→→→   ∨                   │
│           ╱ ╲                                          │
│          ╱   ╲                                         │
│         ╱     ╲                                        │
│                                                         │
│     量を出す              選ぶ・絞る                    │
│     判断しない            基準を持つ                    │
└─────────────────────────────────────────────────────────┘
```

---

## 1. ブレインストーミング

### 1.1 7つのルール

| ルール | 説明 | よくある違反 |
|--------|------|-------------|
| **1. 判断を保留** | 批判・評価しない | 「それは無理だよ」 |
| **2. 大胆なアイデア歓迎** | 実現性は後で考える | 「現実的に考えて...」 |
| **3. 他人のアイデアに乗る** | Yes, And... | 「でも」「しかし」 |
| **4. 一度に一人** | 話を遮らない | 同時発言 |
| **5. 視覚化する** | 絵やスケッチで | 言葉だけ |
| **6. 量を追求** | 目標数を設定 | 5個で満足 |
| **7. トピックに集中** | 脱線しない | 雑談 |

### 1.2 進行の流れ

```markdown
## 準備（5分）
- HMW質問を壁に貼る
- 付箋、ペンを配布
- ルールを確認

## ウォームアップ（5分）
- 関係ないお題で練習
- 「レンガの使い方を100個」など

## 本番（15-30分）
- 1人1枚ずつ付箋に書く
- 書いたら声に出して貼る
- 他の人のアイデアに乗る

## 整理（10分）
- 似たアイデアをグルーピング
- グループに名前をつける
```

### 1.3 ファシリテーターの役割

```
DO:
✓ エネルギーを高く保つ
✓ 沈黙を埋める質問を用意
✓ 発言の少ない人を促す
✓ ルール違反を優しく指摘
✓ 時間管理

DON'T:
✗ 自分のアイデアを押し付ける
✗ 評価する
✗ 参加者を否定する
```

## 2. Crazy 8s

### 2.1 概要

8分間で8つのアイデアをスケッチする高速発想法。

### 2.2 進め方

```
┌────────┬────────┬────────┬────────┐
│   1    │   2    │   3    │   4    │
│  60秒  │  60秒  │  60秒  │  60秒  │
├────────┼────────┼────────┼────────┤
│   5    │   6    │   7    │   8    │
│  60秒  │  60秒  │  60秒  │  60秒  │
└────────┴────────┴────────┴────────┘

1. A4紙を8つに折る
2. タイマーを8分にセット
3. 1分ごとに1マスにスケッチ
4. 時間が来たら次のマスへ（強制）
5. 8分後、全員でシェア
```

### 2.3 コツ

- **考えすぎない**: 手を動かす
- **絵が下手でもOK**: 伝わればいい
- **バリエーション**: 同じアイデアの変形もOK
- **最後が勝負**: 6-8番目に良いアイデアが出やすい

## 3. SCAMPER

### 3.1 概要

既存のアイデアを7つの視点で変形させる発想法。

### 3.2 7つの視点

| 文字 | 視点 | 質問例 |
|------|------|--------|
| **S** | Substitute（代用） | 何を他のもので置き換えられるか？ |
| **C** | Combine（結合） | 何と組み合わせられるか？ |
| **A** | Adapt（適応） | 他から取り入れられるものは？ |
| **M** | Modify（修正） | 大きさ、形、色を変えたら？ |
| **P** | Put to other uses（転用） | 他の目的に使えるか？ |
| **E** | Eliminate（削除） | 何を取り除けるか？ |
| **R** | Reverse/Rearrange（逆転） | 順序や役割を逆にしたら？ |

### 3.3 使い方の例

```
対象: 「コーヒーショップ」

S: 店員をロボットに置き換えたら？
C: 図書館と組み合わせたら？
A: ホテルのコンシェルジュサービスを取り入れたら？
M: 店を10倍大きく/小さくしたら？
P: 夜はバーとして使えないか？
E: レジをなくしたら？
R: お客がコーヒーを作る側になったら？
```

## 4. ブレインライティング

### 4.1 概要

静かに書いて回す、内向型にも優しい発想法。

### 4.2 6-3-5法

```
6人が参加
3つずつアイデアを書く
5分で隣に回す

┌─────────────────────────────────────┐
│  アイデア1  │  アイデア2  │  アイデア3  │
├─────────────────────────────────────┤
│            │            │            │  ← 1人目
├─────────────────────────────────────┤
│            │            │            │  ← 2人目（発展）
├─────────────────────────────────────┤
│            │            │            │  ← 3人目（発展）
├─────────────────────────────────────┤
│            │            │            │  ← 4人目（発展）
├─────────────────────────────────────┤
│            │            │            │  ← 5人目（発展）
├─────────────────────────────────────┤
│            │            │            │  ← 6人目（発展）
└─────────────────────────────────────┘

6ラウンド × 3アイデア × 6人 = 108アイデア
```

## 5. 逆転発想（Reverse Brainstorming）

### 5.1 概要

「最悪にするには？」を考え、逆転させる発想法。

### 5.2 進め方

```
1. 問いを逆転させる
   「どうすれば顧客満足度を上げられるか？」
   → 「どうすれば顧客を怒らせられるか？」

2. 逆の答えを出す
   ・待たせる
   ・無視する
   ・たらい回しにする
   ・約束を破る

3. 逆転させる
   ・待たせない → 即時対応チャット
   ・無視しない → 必ず名前で呼ぶ
   ・たらい回ししない → 一人で完結
   ・約束を守る → 進捗を自動通知
```

## 6. アナロジー発想

### 6.1 概要

他の業界・分野の成功例を借りてくる発想法。

### 6.2 進め方

```
1. 課題を抽象化する
   「予約の待ち時間を減らしたい」
   → 「待ち時間の問題を解決した例は？」

2. 他業界の例を探す
   ・ディズニー（待ち時間のエンタメ化）
   ・Uber（リアルタイムの進捗表示）
   ・病院（番号呼び出しシステム）

3. 自分の文脈に適用する
   ・待ち時間にゲームを提供
   ・順番が来るまでの時間を可視化
   ・順番が近づいたら通知
```

## 7. アイデアの選定

### 7.1 ドット投票（Dot Voting）

```
1. 全アイデアを壁に貼る
2. 各自3〜5個のドットシール（または丸印）を持つ
3. 良いと思うアイデアにシールを貼る
4. 1つに複数シールを貼ってもOK
5. 票の多いものを選出
```

### 7.2 2x2マトリクス

```
              高インパクト
                   │
        ┌──────────┼──────────┐
        │   Big    │   Quick  │
        │   Bet    │   Win    │
高コスト──┼──────────┼──────────┼──低コスト
        │  Money   │   Fill   │
        │   Pit    │   In     │
        └──────────┼──────────┘
                   │
              低インパクト

Quick Win: 優先的に実施
Big Bet: 戦略的に検討
Fill In: 余裕があれば
Money Pit: 避ける
```

### 7.3 NUF テスト

| 基準 | 質問 | スコア |
|------|------|--------|
| **N**ew（新しい） | 他にない独自性があるか？ | 1-5 |
| **U**seful（有用） | ユーザーの問題を解決するか？ | 1-5 |
| **F**easible（実現可能） | 実現できるか？ | 1-5 |

## 8. 成果物のまとめ

| 成果物 | 目的 | 形式 |
|--------|------|------|
| アイデアリスト | 発散結果の記録 | 付箋、写真 |
| グルーピング結果 | アイデアの整理 | アフィニティダイアグラム |
| 選定アイデア（Top 3-5） | 次フェーズへの入力 | 短い説明文 |
| 投票結果 | 優先順位の記録 | スコアリングシート |
